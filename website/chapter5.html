5. جمل if

الاختبارات الشرطية، if-elif-else، القيم المنطقية، والتعامل مع القوائم

في هذا الفصل، ستتعلم كيفية كتابة جمل شرطية تسمح لبرنامجك باتخاذ قرارات بناءً على قيم معينة. باستخدام جمل if، يمكنك التحقق من حالة معينة وتنفيذ كود محدد إذا تحققت تلك الحالة. ستتعلم أيضاً كيفية اختبار شروط متعددة، التعامل مع القيم المنطقية (Boolean)، واستخدام جمل if مع القوائم لإنشاء برامج تفاعلية وقوية.

مثال بسيط

لنفترض أن لدينا قائمة بأسماء السيارات ونريد طباعة أسماء السيارات التي تبدأ بحرف 'b' فقط:

cars = ['bmw', 'audi', 'toyota', 'subaru'] for car in cars: if car == 'bmw': print(car.upper()) else: print(car.title())
📖 شرح الكود: يبدأ الكود بتعريف قائمة بأسماء سيارات. في حلقة for، يتم تمرير كل اسم سيارة واختباره: إذا كان الاسم 'bmw' يُطبع بالأحرف الكبيرة باستخدام upper()، وإلا يُطبع بحرف كبير أول فقط باستخدام title(). المخرجات ستكون BMW و Audi و Toyota و Subaru.
# المخرجات: BMW Audi Toyota Subaru

في هذا المثال، تتحقق جملة if من قيمة كل سيارة. إذا كانت car == 'bmw'، فإنها تطبع الاسم بالأحرف الكبيرة. وإلا، تطبعه بحرف كبير أول.

الاختبارات الشرطية (Conditional Tests)

في قلب كل جملة if يوجد تعبير شرطي (conditional test) يتم تقييمه إلى True أو False. إذا كان التعبير True، يتم تنفيذ الكود الموجود داخل جملة if. إذا كان False، يتم تجاهله.

اختبار المساواة (Equality)

تستخدم علامة == للتحقق من المساواة:

car = 'bmw' print(car == 'bmw') # True print(car == 'audi') # False
📖 شرح الكود: يوضّح هذا الكود كيفية استخدام عامل المساواة == لمقارنة القيم. المتغير car يُسند إليه النص 'bmw'، ثم يُقارن مع 'bmw' فيعيد True، ويُقارن مع 'audi' فيعيد False لأن القيمتين مختلفتان. المخرجات ستكون True ثم False في سطرين منفصلين.

بايثون حساسة لحالة الأحرف عند التحقق من المساواة. 'Bmw' لا تساوي 'bmw'. إذا كنت لا تريد أن تفرق بين الأحرف الكبيرة والصغيرة، استخدم الدالة lower():

car = 'Bmw' print(car.lower() == 'bmw') # True
📖 شرح الكود: يوضح هذا المثال أن بايثون حساسة لحالة الأحرف عند المقارنة، لذلك تُستخدم الدالة lower() لتحويل النص إلى أحرف صغيرة قبل المقارنة. المتغير car يحمل القيمة 'Bmw'، وعند تطبيق lower() يصبح 'bmw' وهو ما يساوي النص المُقارَن. المخرج سيكون True.

اختبار عدم المساواة (Inequality)

تستخدم علامة != للتحقق من عدم المساواة:

requested_topping = 'mushrooms' if requested_topping != 'anchovies': print("Hold the anchovies!")
📖 شرح الكود: هنا يُستخدم عامل عدم المساواة != للتحقق من أن القيمة المطلوبة لا تساوي 'anchovies'. إذا كان الشرط صحيحاً (أي أن الأنشوجة غير مطلوبة)، تُطبَع رسالة تأكيد. بما أن المتغير يحمل 'mushrooms' وهي مختلفة عن 'anchovies'، سيتم تنفيذ الطباعة. المخرج سيكون: Hold the anchovies!
# المخرجات: Hold the anchovies!

المقارنات الرقمية

يمكنك مقارنة الأرقام باستخدام ==, !=, <, <=, >, >=:

age = 18 print(age == 18) # True print(age != 18) # False print(age < 21) # True print(age <= 18) # True print(age > 21) # False print(age >= 18) # True
📖 شرح الكود: يشرح هذا الكود جميع عوامل المقارنة الرقمية في بايثون: المساواة ==، عدم المساواة !=، أصغر من <، أصغر أو يساوي <=، أكبر من >، أكبر أو يساوي >=. المتغير age يساوي 18، لذلك كل اختبار يُعيد إما True أو False حسب العلاقة الرياضية. المخرجات ستكون: True - False - True - True - False - True على التوالي.

اختبار شروط متعددة باستخدام and و or

يمكنك اختبار شرطين أو أكثر في نفس الوقت باستخدام and (يعيد True فقط إذا تحقق الشرطان معاً) or (يعيد True إذا تحقق شرط واحد على الأقل):

age_0 = 22 age_1 = 18 print(age_0 >= 21 and age_1 >= 21) # False print(age_0 >= 21 or age_1 >= 21) # True
📖 شرح الكود: يوضح هذا الكود كيفية دمج شرطين باستخدام and و or. مع and يجب أن يتحقق كلا الشرطين ليعيد True — هنا age_1 أقل من 21 لذا النتيجة False. مع or يكفي تحقق شرط واحد — age_0 >= 21 صحيح لذا النتيجة True. المخرجات ستكون: False ثم True.

التحقق من وجود قيمة في قائمة (in)

استخدم in للتحقق مما إذا كانت قيمة معينة موجودة في قائمة:

requested_toppings = ['mushrooms', 'onions', 'pineapple'] print('mushrooms' in requested_toppings) # True print('pepperoni' in requested_toppings) # False
📖 شرح الكود: يستخدم هذا الكود العامل in للتحقق من وجود قيمة داخل قائمة. القائمة تحتوي على ثلاثة عناصر، والتحقق الأول من وجود 'mushrooms' يُعيد True لأنه موجود، والتحقق الثاني من وجود 'pepperoni' يُعيد False لأنه غير موجود. المخرجات ستكون: True ثم False.

التحقق من عدم وجود قيمة في قائمة (not in)

استخدم not in للتحقق من أن قيمة معينة غير موجودة في قائمة:

banned_users = ['ahmed', 'sara'] user = 'khaled' if user not in banned_users: print(f"{user.title()} يمكنه المشاركة!")
📖 شرح الكود: هذا المثال يستخدم العامل not in للتأكد من أن المستخدم الحالي ليس ضمن قائمة المستخدمين المحظورين. المتغير user = 'khaled' غير موجود في قائمة banned_users، لذلك يتحقق الشرط وتُطبَع رسالة ترحيب مع تنسيق الاسم باستخدام f-string و title(). المخرج سيكون: Khaled يمكنه المشاركة!
# المخرجات: Khaled يمكنه المشاركة!

التعبيرات المنطقية (Boolean Expressions)

القيمة المنطقية (Boolean) هي قيمة إما True أو False. تُستخدم عادةً لتسجيل حالة شيء ما في البرنامج:

game_active = True can_edit = False
📖 شرح الكود: يوضح هذا الكود ماهية القيم المنطقية (Boolean) التي تأخذ إحدى قيمتين فقط: True أو False. المتغير game_active يشير إلى أن اللعبة نشطة، والمتغير can_edit يشير إلى أن التحرير غير مسموح. هذه القيم تجعل الشروط أكثر وضوحاً وكفاءة من استخدام نصوص مثل 'active' أو 'inactive'.

القيم المنطقية تجعل الكود أكثر وضوحاً. بدلاً من تخزين 'active' أو 'inactive' كنص، استخدم True أو False لتكون أكثر دقة وسهولة في الاختبار.

جمل if

هناك عدة أنواع من جمل if في بايثون، ولكل منها استخداماتها المناسبة.

جملة if بسيطة

أبسط شكل لجملة if هو اختبار شرط وتنفيذ كود إذا تحقق:

age = 19 if age >= 18: print("أنت في سن legal للتصويت!")
📖 شرح الكود: أبسط شكل لجملة if: يتم اختبار شرط واحد فقط، وإذا تحقق يُنفَّذ الكود الذي يليه. المتغير age = 19 وهو أكبر من أو يساوي 18، لذلك يتحقق الشرط وتُطبَع الرسالة. إذا كان العمر أقل من 18، لما ظهر أي مخرج على الإطلاق.
# المخرجات: أنت في سن legal للتصويت!

جملة if-else

استخدم if-else لتنفيذ كود عندما يتحقق الشرط، وكود آخر عندما لا يتحقق:

age = 17 if age >= 18: print("يمكنك التصويت!") else: print("عذراً، أنت أصغر من سن التصويت.")
📖 شرح الكود: جملة if-else توفر مسارين: إذا تحقق الشرط يُنفَّذ كود if، وإلا يُنفَّذ كود else. هنا العمر 17 وهو أقل من 18، لذلك لا يتحقق الشرط الأول ويُنتقل مباشرة إلى else. المخرج سيكون: عذراً، أنت أصغر من سن التصويت.
# المخرجات: عذراً، أنت أصغر من سن التصويت.

سلسلة if-elif-else

عندما تحتاج إلى اختبار أكثر من شرطين، استخدم سلسلة if-elif-else. تختبر بايثون كل شرط بدورها، وعندما تجد شرطاً يتحقق، تنفذ الكود المرتبط به وتتخطى الباقي:

age = 12 if age < 4: print("الدخول مجاني!") elif age < 18: print("سعر التذكرة 25 ريالاً.") else: print("سعر التذكرة 40 ريالاً.")
📖 شرح الكود: سلسلة if-elif-else تختبر شروطاً متعددة بالترتيب وتتوقف عند أول شرط يتحقق. العمر 12 لا يحقق الشرط الأول (أقل من 4)، لكنه يحقق الشرط الثاني (أقل من 18)، لذلك تُطبَع رسالة سعر التذكرة 25 ريالاً وتتخطى باقي الشروط. المخرج سيكون: سعر التذكرة 25 ريالاً.
# المخرجات: سعر التذكرة 25 ريالاً.

استخدام عدة كتل elif

يمكنك استخدام أي عدد تريده من كتل elif:

age = 65 if age < 4: print("الدخول مجاني!") elif age < 18: print("سعر التذكرة 25 ريالاً.") elif age < 65: print("سعر التذكرة 40 ريالاً.") else: print("سعر التذكرة 20 ريالاً (خصم كبار السن).")
📖 شرح الكود: هنا نستخدم كتلتين من elif لتوفير المزيد من الخيارات. العمر 65 لا يحقق أي شرط من الشروط السابقة (ليس أقل من 4 أو 18 أو 65)، فينتقل إلى else الذي يُطبِّق خصماً لكبار السن. المخرج سيكون: سعر التذكرة 20 ريالاً (خصم كبار السن).
# المخرجات: سعر التذكرة 20 ريالاً (خصم كبار السن).

حذف else

بايثون لا تتطلب وجود كتلة else في نهاية سلسلة if-elif. أحياناً يكون من الأفضل استخدام elif إضافي بدلاً من else لتكون أكثر وضوحاً في التعامل مع الحالات المحددة:

age = 65 if age < 4: price = 0 elif age < 18: price = 25 elif age < 65: price = 40 elif age >= 65: price = 20 print(f"سعر تذكرتك هو {price} ريالاً.")
📖 شرح الكود: هذا المثال يوضح أنه يمكن حذف else واستخدام elif إضافي بدلاً منه لجعل النية أكثر وضوحاً. بدلاً من else، تم استخدام elif age >= 65 لبيان أن هذا الشرط مخصص لكبار السن بشكل صريح. المخرج سيكون: سعر تذكرتك هو 20 ريالاً.

استخدام elif age >= 65 بدلاً من else يجعل الكود أكثر وضوحاً ويظهر نيتك بشكل صريح.

اختبار شروط متعددة

جمل if-elif-else تختبر شرطاً واحداً فقط في كل مرة وتتوقف عند أول شرط يتحقق. لكن في بعض الأحيان قد تحتاج لاختبار جميع الشروط الممكنة (كل منها على حدة)، بغض النظر عما إذا كانت الشروط الأخرى قد تحققت. في هذه الحالة، استخدم عدة جمل if مستقلة:

requested_toppings = ['mushrooms', 'extra cheese'] if 'mushrooms' in requested_toppings: print("Adding mushrooms.") if 'pepperoni' in requested_toppings: print("Adding pepperoni.") if 'extra cheese' in requested_toppings: print("Adding extra cheese.") print("\nYour pizza is ready!")
📖 شرح الكود: استخدام جمل if مستقلة يضمن اختبار كل شرط بغض النظر عن الشروط الأخرى. القائمة تحتوي على mushrooms و extra cheese فقط، لذلك تُنفَّذ أول وآخر جملة if فقط، بينما تتخطى جملة pepperoni لأن الشرط لم يتحقق. المخرجات ستكون: Adding mushrooms. و Adding extra cheese. و Your pizza is ready!
# المخرجات: Adding mushrooms. Adding extra cheese. Your pizza is ready!

إذا استخدمنا if-elif بدلاً من ثلاث جمل if منفصلة، لما تمت إضافة الجبن الإضافي لأن بايثون ستتوقف عند أول شرط يتحقق.

💡 ملاحظة مهمة: استخدم سلسلة if-elif-else عندما تحتاج إلى شرط واحد فقط للتنفيذ. استخدم عدة جمل if مستقلة عندما تريد اختبار جميع الشروط بغض النظر عن بعضها البعض.

استخدام if مع القوائم

يوفر دمج جمل if مع القوائم مرونة كبيرة في البرمجة. إليك بعض السيناريوهات الشائعة.

التحقق من عناصر خاصة

يمكنك التعامل مع عناصر محددة في القائمة بشكل خاص:

requested_toppings = ['mushrooms', 'green peppers', 'extra cheese'] for requested_topping in requested_toppings: if requested_topping == 'green peppers': print("Sorry, we are out of green peppers right now.") else: print(f"Adding {requested_topping}.") print("\nFinished making your pizza!")
📖 شرح الكود: هذا المثال يدمج حلقة for مع if-else للتعامل مع عناصر القائمة بشكل مختلف. في كل دورة من الحلقة، يُختبر العنصر: إذا كان 'green peppers' (غير متوفر) تُطبَع رسالة اعتذار، وإلا تُطبَع رسالة إضافة. المخرجات ستظهر ثلاث رسائل ثم رسالة الانتهاء.
# المخرجات: Adding mushrooms. Sorry, we are out of green peppers right now. Adding extra cheese. Finished making your pizza!

التحقق من أن القائمة ليست فارغة

عند استخدام اسم قائمة في شرط if، تعيد بايثون True إذا كانت القائمة تحتوي على عنصر واحد على الأقل، و False إذا كانت فارغة:

requested_toppings = [] if requested_toppings: for requested_topping in requested_toppings: print(f"Adding {requested_topping}.") print("\nFinished making your pizza!") else: print("Are you sure you want a plain pizza?")
📖 شرح الكود: استخدام اسم القائمة مباشرة في شرط if هو أسلوب بايثوني للتحقق من أنها ليست فارغة. القائمة فارغة ([])، لذلك يُعامِلُها الشرط كـ False ويُنتقل إلى else. هذا أفضل من استخدام len() لأنه أكثر إيجازاً ووضوحاً. المخرج سيكون: Are you sure you want a plain pizza?
# المخرجات: Are you sure you want a plain pizza?

مقارنة قوائم متعددة

يمكنك استخدام القوائم وجمل if للتحقق من توفر عناصر في قائمة أخرى:

available_toppings = ['mushrooms', 'olives', 'green peppers', 'pepperoni', 'pineapple', 'extra cheese'] requested_toppings = ['mushrooms', 'french fries', 'extra cheese'] for requested_topping in requested_toppings: if requested_topping in available_toppings: print(f"Adding {requested_topping}.") else: print(f"Sorry, we don't have {requested_topping}.") print("\nFinished making your pizza!")
📖 شرح الكود: هذا المثال يقارن بين قائمتين: الإضافات المتاحة والإضافات المطلوبة. الحلقة تتحقق من كل عنصر مطلوب: إذا كان موجوداً في قائمة المتاح يُضاف إلى البيتزا، وإلا تظهر رسالة اعتذار. بما أن 'french fries' غير متوفرة، ستظهر رسالة اعتذار عنها. المخرجات ستظهر رسالتَي إضافة ورسالة اعتذار واحدة.
# المخرجات: Adding mushrooms. Sorry, we don't have french fries. Adding extra cheese. Finished making your pizza!

هذا النوع من الكود قوي جداً — يسمح لبرنامجك بالتحقق من البيانات المتاحة قبل اتخاذ إجراء، مما يمنع الأخطاء ويحسن تجربة المستخدم.

التمارين

تمرين 5-1: الاختبارات الشرطية
اكتب سلسلة من الاختبارات الشرطية. أنشئ متغيراً واختبر قيمته باستخدام == و != و < و > و <= و >=. اطبع عبارة تصف كل اختبار ونتيجته.
تمرين 5-2: المزيد من الاختبارات الشرطية
اختبر المساواة وعدم المساواة مع السلاسل النصية. اختبر باستخدام lower(). اختبر using and و or. اختبر باستخدام in و not in.
تمرين 5-3: ألوان الفضائيين #1
أنشئ متغيراً باسم alien_color واجعل قيمته 'green'. اكتب جملة if تتحقق إذا كان لون الفضائي أخضر واطبع "اللاعب ربح 5 نقاط". اكتب نسخة أخرى تفشل (لا يوجد مخرجات).
تمرين 5-4: ألوان الفضائيين #2
اختر لوناً للفضائي. اكتب جملة if-else: إذا كان أخضر اطبع "ربحت 5 نقاط لإصابتك الفضائي". وإلا اطبع "ربحت 10 نقاط".
تمرين 5-5: ألوان الفضائيين #3
حوّل تمرين 5-4 إلى سلسلة if-elif-else بثلاثة ألوان: أخضر (5 نقاط)، أصفر (10 نقاط)، أحمر (15 نقطة).
تمرين 5-6: مراحل العمر
اكتب سلسلة if-elif-else لتحديد مرحلة العمر: رضيع (أقل من 2)، طفل (أقل من 13)، مراهق (أقل من 20)، بالغ (أقل من 65)، كبير سن (65 أو أكثر).
تمرين 5-7: الفاكهة المفضلة
أنشئ قائمة favorite_fruits. اكتب 5 جمل if تتحقق إذا كانت فاكهة معينة موجودة في القائمة.
تمرين 5-8: الترحيب بالمستخدمين
أنشئ قائمة بأسماء المستخدمين (بما فيها 'admin'). اكتب حلقة ترحب بكل مستخدم، واطبع رسالة خاصة للإدمن.
تمرين 5-9: لا مستخدمين
أضف اختباراً في تمرين 5-8 ليتأكد أن قائمة المستخدمين ليست فارغة. إذا كانت فارغة، اطبع "نحتاج إلى بعض المستخدمين!"

5.5 pattern matching - match/case (Python 3.10+)

منذ Python 3.10، بايثون تدعم مطابقة الأنماط الهيكلية (Structural Pattern Matching)، مشابهة لـ switch/case في اللغات الأخرى لكن أقوى بكثير:

# مطابقة بسيطة مثل switch def describe_status(code): match code: case 200: return 'OK' case 404: return 'Not Found' case 500: return 'Server Error' case _: # wildcard (افتراضي) return 'Unknown' print(describe_status(404)) # Not Found
📖 شرح الكود: هذا المثال يُقدِّم match/case وهو نمط مطابقة الأنماط الهيكلية المتاح من Python 3.10+. الدالة describe_status تستقبل رمز حالة وتُطابقه مع أنماط مختلفة: إذا كان 200 تُعيد 'OK'، إذا كان 404 تُعيد 'Not Found'، وإلا تستخدم wildcard (_) كحالة افتراضية. المخرج سيكون: Not Found.
# مطابقة مع تفكيك (destructuring) def process_point(point): match point: case (0, 0): return 'Origin' case (0, y): return f'On Y axis at {y}' case (x, 0): return f'On X axis at {x}' case (x, y): return f'Point({x}, {y})' case _: return 'Not a point' print(process_point((0, 5))) # On Y axis at 5
📖 شرح الكود: هذا المثال يوضح قدرة match/case على تفكيك القيم المركبة (tuple destructuring). النقطة (0, 5) تُطابِق الحالة الثانية (0, y) حيث x = 0 و y = 5، مما يعني أنها تقع على المحور Y. المخرج سيكون: On Y axis at 5.
# مطابقة مع شروط إضافية (guards) def classify(value): match value: case x if x < 0: return 'Negative' case x if x == 0: return 'Zero' case x if x > 0: return 'Positive' # مطابقة مع قواميس def handle_command(cmd): match cmd: case {'action': 'move', 'x': x, 'y': y}: return f'Moving to ({x}, {y})' case {'action': 'stop'}: return 'Stopping' case _: return 'Unknown command' print(handle_command({'action': 'move', 'x': 10, 'y': 20}))
📖 شرح الكود: هذا المثال يعرِض ميزتين متقدمتين في match/case: الشروط الإضافية (guards) باستخدام if داخل case لتصنيف الأرقام، ومطابقة القواميس (dict matching) حيث تُستخرَج المفاتيح والقيم. الدالة handle_command تستقبل قاموس أوامر وتستخرج الإحداثيات إذا كان الأمر move. المخرج سيكون: Moving to (10, 20).
💡 من Fluent Python: match/case ليس مجرد switch — هو مطابقة أنماط هيكلية يمكنها تفكيك القيم، مطابقة الشكل، وتطبيق شروط. مثالي لمعالجة ASTs، نظام الأوامر، وتحليل JSON. استخدم case _: كحالة افتراضية دائماً.

Enum - ثوابت آمنة وقابلة للقراءة

وحدة enum توفّر ثوابت مسماة آمنة — بديل أفضل للمتغيرات العامة (global constants) أو القيم السحرية (magic numbers):

from enum import Enum, auto class Color(Enum): RED = 'red' GREEN = 'green' BLUE = 'blue' class Status(Enum): PENDING = auto() # قيم تلقائية 1, 2, 3... ACTIVE = auto() DONE = auto() # استخدام — مقارنة بالاسم وليس القيمة def handle(color): match color: # enum يتكامل مع match/case! case Color.RED: return 'Stop' case Color.GREEN: return 'Go' case _: return 'Unknown' print(handle(Color.RED)) # Stop print(Status.ACTIVE.name) # ACTIVE print(Status.ACTIVE.value) # 2
📖 شرح الكود: هذا المثال يشرح وحدة Enum لإنشاء ثوابت مسماة آمنة وقابلة للقراءة. Color يُعرِّف ثلاثة ألوان بقيم نصية، بينما Status يستخدم auto() لتوليد قيم رقمية تلقائياً. يتكامل Enum مع match/case بشكل مثالي لمقارنة الثوابت بطريقة آمنة من الأخطاء الإملائية. المخرجات ستكون: Stop - ACTIVE - 2.
💡 من Fluent Python: استخدم Enum بدلاً من السلاسل النصية أو الأرقام للثوابت — يمنع الأخطاء الإملائية ويدعم match/case. auto() يعيّن قيماً تلقائية. @unique يمنع تكرار القيم.

ملخص الفصل

📋 ملخص الفصل 5: جمل if

  • تعلمت كيفية كتابة اختبارات شرطية باستخدام ==, !=, <, >, <=, >=
  • تعرفت على اختبار شروط متعددة باستخدام and و or
  • تعلمت التحقق من وجود قيمة في قائمة باستخدام in و not in
  • تعرفت على القيم المنطقية (Boolean) True و False
  • أتقنت استخدام جمل if البسيطة، if-else، و if-elif-else
  • عرفت متى تستخدم عدة elif ومتى تحذف else
  • تعلمت الفرق بين if-elif-else وجمل if المتعددة
  • تعاملت مع القوائم داخل جمل if: عناصر خاصة، قوائم فارغة، ومقارنة قوائم
  • طورت كوداً يتفاعل مع البيانات بشكل ذكي ويتجنب الأخطاء
💡 نصيحة ذهبية: استفد من Falsy values في بايثون: 0، ''، []، {}، None كلها تعامل كـ False. اكتب if not items: بدلاً من if len(items) == 0:. استخدم x if condition else y (Ternary) للشروط البسيطة في سطر واحد.
⚠️ خطأ شائع: استخدام == لمقارنة القيم مع None. استخدم is None و is not None بدلاً من ذلك. وكذلك if x: للتحقق من truthy/falsy بدلاً من if x == True:.
🏋️ تمرين: اكتب برنامج تقدير درجات: اطلب من المستخدم رقم بين 0 و 100، ثم اطبع التقدير: 90+ = A، 80+ = B، 70+ = C، 60+ = D، أقل = F. استخدم if-elif-else. تأكد من التعامل مع القيم خارج النطاق.
