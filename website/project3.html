الفصل 18: البدء مع Django

18.1 تثبيت Django وإنشاء المشروع

Django هو إطار عمل (framework) لتطوير تطبيقات الويب بلغة بايثون. يتبع نمط MVT (Model-View-Template).

# تثبيت Django python3 -m pip install django # إنشاء مشروع جديد django-admin startproject learning_log .

الأمر startproject ينشئ هيكل المشروع الأساسي. النقطة ( . ) في النهاية تعني إنشاء المشروع في المجلد الحالي.

18.2 هيكل مشروع Django

بعد إنشاء المشروع، سترى الملفات التالية:

learning_log/ ├── manage.py # أداة سطر الأوامر لإدارة المشروع └── learning_log/ ├── __init__.py # يجعل المجلد حزمة Python ├── settings.py # إعدادات المشروع الرئيسية ├── urls.py # تعريف مسارات URL ├── asgi.py # لتشغيل التطبيق بخوادم ASGI └── wsgi.py # لتشغيل التطبيق بخوادم WSGI

18.3 إنشاء تطبيق (App) داخل المشروع

مشروع Django يتكون من تطبيقات متعددة. كل تطبيق يؤدي وظيفة محددة. لننشئ تطبيقاً باسم learning_logs:

python3 manage.py startapp learning_logs

18.4 تعريف النماذج (Models)

النماذج تمثل هيكل البيانات في قاعدة البيانات. كل Model يُترجم إلى جدول في قاعدة البيانات. لتطبيقنا نموذجان: Topic (الموضوع) و Entry (الإدخال):

from django.db import models class Topic(models.Model): text = models.CharField(max_length=200) date_added = models.DateTimeField(auto_now_add=True) def __str__(self): return self.text class Entry(models.Model): topic = models.ForeignKey(Topic, on_delete=models.CASCADE) text = models.TextField() date_added = models.DateTimeField(auto_now_add=True) class Meta: verbose_name_plural = 'entries' def __str__(self): return self.text[:50] + "..."

CharField لحقول النص القصير، TextField للنص الطويل، ForeignKey لعلاقة "واحد إلى متعدد" بين Topic و Entry. on_delete=models.CASCADE يعني: إذا حُذف الموضوع، احذف جميع الإدخالات المرتبطة به.

18.5 ترحيل قاعدة البيانات (Migrations)

بعد تعريف النماذج، نحتاج إلى إنشاء جداولها في قاعدة البيانات:

python3 manage.py makemigrations python3 manage.py migrate

makemigrations ينشئ ملفات الترحيل استناداً إلى تغييرات النماذج. migrate يطبق هذه التغييرات على قاعدة البيانات.

18.6 لوحة التحكم (Admin)

Django يوفر لوحة تحكم مدمجة لإدارة البيانات. أولاً، أنشئ مستخدم مشرف (superuser):

python3 manage.py createsuperuser

ثم سجّل النماذج في ملف admin.py:

from django.contrib import admin from .models import Topic, Entry admin.site.register(Topic) admin.site.register(Entry)

الآن يمكنك الدخول إلى http://localhost:8000/admin/ وإدارة بياناتك.

18.7 المناظرات (Views) والمسارات (URLs)

المناظرات هي دوال بايثون تستقبل طلب HTTP وتعيد استجابة HTTP. غالباً ما تكون استجابة HTML. نستخدم render() لعرض قالب HTML:

from django.shortcuts import render def index(request): return render(request, 'learning_logs/index.html') def topics(request): topics = Topic.objects.order_by('date_added') context = {'topics': topics} return render(request, 'learning_logs/topics.html', context) def topic(request, topic_id): topic = Topic.objects.get(id=topic_id) entries = topic.entry_set.order_by('-date_added') context = {'topic': topic, 'entries': entries} return render(request, 'learning_logs/topic.html', context)

نربط هذه المناظرات بعناوين URL عبر ملف urls.py:

from django.urls import path from . import views urlpatterns = [ path('', views.index, name='index'), path('topics/', views.topics, name='topics'), path('topics/<int:topic_id>/', views.topic, name='topic'), ]

18.8 القوالب (Templates)

القوالب هي ملفات HTML تحتوي على علامات Django الخاصة. مثال لقالب الموضوعات:

{% extends 'learning_logs/base.html' %} {% block content %} <p>Topics</p> <ul> {% for topic in topics %} <li> <a href="{% url 'learning_logs:topic' topic.id %}"> {{ topic }} </a> </li> {% empty %} <li>No topics have been added yet.</li> {% endfor %} </ul> {% endblock content %}

{% extends %} يستخدم الوراثة في القوالب (template inheritance). {{ }} يعرض قيمة متغير. {% url %} يولّد رابطاً لمسار معين.

ملاحظة: Django يفرض فصل الاهتمامات (separation of concerns): النماذج تتعامل مع البيانات، المناظرات تتعامل مع المنطق، القوالب تتعامل مع العرض. هذا يجعل الكود منظماً وسهل الصيانة.

الفصل 19: حسابات المستخدمين

19.1 السماح للمستخدمين بتسجيل الدخول

Django يأتي مع نظام مصادقة مدمج. أولاً، ننشئ تطبيقاً للمستخدمين:

python3 manage.py startapp accounts

نضيف المسارات الخاصة بتسجيل الدخول والخروج. نستخدم مناظرات Django المدمجة بدلاً من كتابتها من الصفر:

# urls.py لتطبيق accounts from django.urls import path, include from django.contrib.auth import views as auth_views urlpatterns = [ path('login/', auth_views.LoginView.as_view( template_name='registration/login.html'), name='login'), path('logout/', auth_views.LogoutView.as_view(), name='logout'), ]

19.2 قالب تسجيل الدخول

ننشئ قالب HTML بسيط لصفحة تسجيل الدخول:

{% extends 'learning_logs/base.html' %} {% block content %} {% if form.errors %} <p>Your username and password didn't match. Try again.</p> {% endif %} <form method="post" action="{% url 'accounts:login' %}"> {% csrf_token %} {{ form.as_div }} <button name="submit">Log in</button> </form> {% endblock content %}

{% csrf_token %} يحمي النموذج من هجمات CSRF (Cross-Site Request Forgery) — وهي حماية أمنية مدمجة في Django.

19.3 تسجيل المستخدمين الجدد

نحتاج إلى صفحة تسجيل جديدة. ننشئ مناظر register التي تعالج نموذج تسجيل:

from django.shortcuts import render, redirect from django.contrib.auth import login as auth_login from django.contrib.auth.forms import UserCreationForm def register(request): if request.method != 'POST': form = UserCreationForm() else: form = UserCreationForm(data=request.POST) if form.is_valid(): new_user = form.save() auth_login(request, new_user) return redirect('learning_logs:index') context = {'form': form} return render(request, 'registration/register.html', context)

19.4 السماح للمستخدمين بامتلاك بياناتهم

نحتاج إلى ربط كل موضوع بمستخدمه. نضيف حقل owner إلى نموذج Topic:

from django.contrib.auth.models import User class Topic(models.Model): text = models.CharField(max_length=200) date_added = models.DateTimeField(auto_now_add=True) owner = models.ForeignKey(User, on_delete=models.CASCADE)

ثم نعدّل المناظر لترشيح الموضوعات حسب المستخدم الحالي:

from django.contrib.auth.decorators import login_required @login_required def topics(request): topics = Topic.objects.filter(owner=request.user).order_by('date_added') context = {'topics': topics} return render(request, 'learning_logs/topics.html', context)

@login_required يضمن أن المستخدم مسجل الدخول قبل الوصول إلى الصفحة. إذا لم يكن مسجلاً، سيُحوّل إلى صفحة تسجيل الدخول.

الفصل 20: التنسيق والنشر

20.1 تنسيق التطبيق باستخدام Bootstrap

Bootstrap هي مكتبة CSS جاهزة تجعل تطبيقك يبدو احترافياً دون كتابة CSS من الصفر. نثبّت حزمة django-bootstrap5:

python3 -m pip install django-bootstrap5

نضيفها إلى INSTALLED_APPS في ملف settings.py:

INSTALLED_APPS = [ # ... تطبيقات Django الافتراضية 'learning_logs', 'accounts', 'django_bootstrap5', ]

ثم نستخدم علامات Bootstrap في قوالبنا:

{% load django_bootstrap5 %} <form method="post" action="{% url 'accounts:register' %}"> {% csrf_token %} {% django_bootstrap5_form form %} <button name="submit" class="btn btn-primary">Register</button> </form>

20.2 نشر التطبيق (Deployment)

لجعل تطبيقك متاحاً على الإنترنت، تحتاج إلى نشره على خادم ويب. الإصدارات الحديثة من الكتاب تستخدم Platform.sh أو Railway أو PythonAnywhere بدلاً من Heroku (الذي لم يعد يقدم خطة مجانية). إليك الخطوات العامة:

1. إنشاء ملف requirements.txt:

python3 -m pip freeze > requirements.txt

2. إعداد متغيرات البيئة للإعدادات الحساسة:

# learning_log/settings.py import os SECRET_KEY = os.environ.get('DJANGO_SECRET_KEY', 'your-dev-key') DEBUG = os.environ.get('DJANGO_DEBUG', 'True') != 'False' ALLOWED_HOSTS = ['your-app-name.herokuapp.com', 'localhost']

3. إعداد قاعدة البيانات: استخدم dj-database-url لتبديل قاعدة البيانات بسهولة بين SQLite (للتطوير) و PostgreSQL (للإنتاج).

4. رفع الكود ونشره:

# مثال لرفع الكود لـ Railway railway login railway init railway up

20.3 الأمان في الإنتاج

عند نشر التطبيق، هناك إعدادات أمان مهمة:

# learning_log/settings.py — إعدادات الإنتاج DEBUG = False SECURE_SSL_REDIRECT = True SESSION_COOKIE_SECURE = True CSRF_COOKIE_SECURE = True SECURE_HSTS_SECONDS = 31536000 SECURE_HSTS_INCLUDE_SUBDOMAINS = True SECURE_HSTS_PRELOAD = True

هذه الإعدادات تضفر HTTPS (تشفير الاتصالات)، وتعطيل وضع التصحيح، وتفعيل HSTS (HTTP Strict Transport Security) لمنع هجمات الرجل في المنتصف.

المفاهيم الرئيسية

📝 المفاهيم التي تعلمتها في هذا المشروع

  • Django: إطار عمل ويب قوي بلغة بايثون يتبع نمط MVT
  • النماذج (Models): تعريف هيكل قاعدة البيانات باستخدام كلاسات بايثون
  • الترحيلات (Migrations): تطبيق تغييرات النماذج على قاعدة البيانات
  • لوحة التحكم (Admin): واجهة مدمجة لإدارة البيانات
  • المناظرات (Views): دوال تعالج الطلبات وترجع الاستجابات
  • القوالب (Templates): ملفات HTML مع علامات Django ديناميكية
  • نظام المصادقة (Authentication): تسجيل الدخول والخروج باستخدام نظام Django المدمج
  • الـ decorator @login_required: حماية الصفحات من المستخدمين غير المسجلين
  • Bootstrap: مكتبة CSS لتحسين مظهر التطبيق بسرعة
  • النشر (Deployment): رفع التطبيق إلى الإنترنت ليكون متاحاً للجميع
تنبيه: عند نشر التطبيق، تأكد أبداً من عدم تضمين SECRET_KEY أو كلمات مرور قاعدة البيانات في الكود نفسه. استخدم متغيرات البيئة (environment variables) للمعلومات الحساسة.
