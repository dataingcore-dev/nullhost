الملحق أ: تثبيت بايثون على أنظمة التشغيل المختلفة

قبل أن تبدأ رحلتك مع بايثون، تحتاج إلى تثبيتها على جهازك. في هذا الملحق، سنشرح كيفية تثبيت بايثون على أنظمة التشغيل الرئيسية الثلاثة: Windows و macOS و Linux.

ملاحظة: بايثون 3 هي الإصدار المستخدم في هذا الكتاب. تأكد من تثبيت الإصدار 3.6 أو أحدث. لم تعد بايثون 2 مدعومة منذ يناير 2020.

تثبيت بايثون على Windows

تثبيت بايثون على Windows عملية بسيطة. اتبع الخطوات التالية:

  1. اذهب إلى الموقع الرسمي python.org.
  2. حمل أحدث إصدار من بايثون 3 (ستجد زر تحميل أصفر في الصفحة الرئيسية).
  3. شغّل ملف التثبيت .exe الذي حملته.
  4. مهم جداً: حدد الخيار "Add Python to PATH" في أسفل نافذة المثبت قبل النقر على "Install Now". هذا يسمح لك بتشغيل بايثون من سطر الأوامر.
  5. انقر على "Install Now" واتبع التعليمات.
  6. بعد اكتمال التثبيت، افتح موجه الأوامر (Command Prompt) واكتب python --version لتأكيد التثبيت.
C:\Users\YourName> python --version Python 3.12.0

تثبيت بايثون على macOS

نظام macOS يأتي مع بايثون 2 مثبتة مسبقاً، لكننا نحتاج بايثون 3. هناك طريقتان للتثبيت:

الطريقة الأولى: استخدام المثبت الرسمي

  1. اذهب إلى python.org وحمل مثبت macOS.
  2. شغّل ملف .pkg واتبع التعليمات.
  3. بعد التثبيت، افتح Terminal واكتب python3 --version.

الطريقة الثانية: استخدام Homebrew (موصى بها للمطورين)

  1. شغّل Terminal.
  2. إذا لم يكن Homebrew مثبتاً، ثبته بتشغيل الأمر التالي:
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  3. ثم ثبّت بايثون باستخدام الأمر:
    brew install python3
  4. تأكد من التثبيت:
    python3 --version
ملاحظة: على macOS، قد تحتاج إلى كتابة python3 بدلاً من python لتشغيل بايثون 3. وبالمثل، استخدم pip3 بدلاً من pip.

تثبيت بايثون على Linux

معظم توزيعات Linux تأتي مع بايثون 3 مثبتة مسبقاً. يمكنك التحقق عن طريق تشغيل:

python3 --version

إذا لم يكن بايثون 3 مثبتاً، استخدم مدير الحزم الخاص بتوزيعتك:

Ubuntu / Debian:

sudo apt update sudo apt install python3 python3-pip python3-venv

Fedora:

sudo dnf install python3 python3-pip

Arch Linux:

sudo pacman -S python python-pip

تثبيت pip و venv

pip هو مدير حزم بايثون، و venv هو أداة لإنشاء بيئات افتراضية. معظم مثبتات بايثون الحديثة تتضمنهما تلقائياً.

للتحقق من تثبيت pip:

pip --version # Windows pip3 --version # macOS / Linux

لإنشاء بيئة افتراضية وتفعيلها:

# إنشاء البيئة الافتراضية python -m venv myenv # تفعيلها على Windows myenv\Scripts\activate # تفعيلها على macOS / Linux source myenv/bin/activate

تشغيل بايثون لأول مرة

بعد تثبيت بايثون، يمكنك تشغيل أول برنامج لك بطريقتين:

1. الوضع التفاعلي (Interactive Shell): اكتب python (أو python3) في سطر الأوامر:

$ python Python 3.12.0 (default, Oct 16 2023, 12:00:00) [GCC 11.3.0] on linux Type "help", "copyright", "credits" or "license" for more information. >>> print("Hello, World!") Hello, World! >>> exit()

2. تشغيل ملف بايثون: أنشئ ملفاً باسم hello.py بالمحتوى التالي:

print("Hello, World!")

ثم شغّله:

python hello.py

خلاصة الملحق أ

  • حمل بايثون من python.org أو استخدم مدير الحزم الخاص بنظامك.
  • تأكد من تحديد "Add Python to PATH" عند التثبيت على Windows.
  • استخدم python3 على macOS/Linux و python على Windows.
  • تأكد من تثبيت pip واستخدم البيئات الافتراضية (venv) لفصل مشاريعك.

الملحق ب: محررات النصوص

محرر النصوص هو الأداة الأساسية التي ستستخدمها لكتابة أكواد بايثون. في هذا الملحق، سنستعرض أفضل محررات النصوص والبيئات المتكاملة (IDEs) المتاحة، مع التركيز على الخيارات المناسبة للمبتدئين والمحترفين على حد سواء.

ما الذي يميز محرراً جيداً لبايثون؟

المحرر الجيد لبايثون يجب أن يوفر:

  • تظليل الكود (Syntax Highlighting): تلوين الكلمات المفتاحية والأكواد لتسهيل القراءة.
  • الإكمال التلقائي (Autocompletion): اقتراح أسماء الدوال والمتغيرات أثناء الكتابة.
  • فحص الأخطاء (Linting): التنبيه إلى الأخطاء البرمجية أثناء الكتابة.
  • تصحيح الأخطاء (Debugging): إمكانية تشغيل الكود خطوة بخطوة لاكتشاف المشاكل.
  • إدارة المشاريع: القدرة على التعامل مع مجلدات وملفات متعددة.

VS Code (الخيار الموصى به للمبتدئين)

Visual Studio Code هو محرر نصوص مجاني ومفتوح المصدر من Microsoft. يعتبر الخيار الأكثر شعبية لبرمجة بايثون حالياً.

مميزاته:

  • مجاني بالكامل وخفيف الوزن.
  • متوفر على Windows و macOS و Linux.
  • مكتبة ضخمة من الإضافات (Extensions).
  • تكامل ممتاز مع Git والتحكم بالإصدارات.
  • واجهة مستخدم عربية (يمكن تغيير اللغة من لوحة الإضافات).

تثبيت VS Code:

  1. اذهب إلى code.visualstudio.com.
  2. حمل المثبت المناسب لنظام تشغيلك.
  3. شغّل المثبت واتبع التعليمات.

إعداد VS Code لبايثون:

  1. افتح VS Code.
  2. اضغط على أيقونة الإضافات (Extensions) في الشريط الجانبي الأيسر أو استخدم الاختصار Ctrl+Shift+X.
  3. ابحث عن "Python" وثبّت الإضافة الرسمية من Microsoft (Python Extension Pack).
  4. افتح ملف بايثون (.py) أو أنشئ مشروعاً جديداً.
  5. لتشغيل الكود، اضغط على زر المثلث الأخضر (▶️) في أعلى يمين المحرر أو استخدم Ctrl+F5.
ملاحظة: VS Code سيطلب منك تحديد مفسر بايثون (Python Interpreter) عند فتح أول ملف بايثون. اختر الإصدار الذي ثبته في الملحق أ.

PyCharm (بيئة تطوير متكاملة احترافية)

PyCharm هو بيئة تطوير متكاملة (IDE) متخصصة في بايثون من شركة JetBrains. يتوفر بنسختين: مجانية (Community) ومدفوعة (Professional).

مميزاته:

  • محرر بايثون متقدم مع إكمال ذكي للكود (Intelligent Code Completion).
  • مصحح أخطاء قوي جداً (Debugger).
  • أدوات متكاملة لاختبار الكود وتغطيته.
  • دعم مدمج لقواعد البيانات (في النسخة المدفوعة).
  • إدارة متقدمة للمشاريع والبيئات الافتراضية.

تثبيت PyCharm:

  1. اذهب إلى jetbrains.com/pycharm.
  2. حمل النسخة Community (المجانية) أو Professional (للاستخدام التجاري).
  3. شغّل المثبت. على Windows، تأكد من تحديد الخيارات "Add to PATH" و "Create Desktop Shortcut".

محررات أخرى

Sublime Text: محرر نصوص سريع وخفيف جداً. مناسب للكتابة السريعة لكنه يتطلب إضافات لبرمجة بايثون. غير مجاني لكن يمكنك استخدام نسخته التقييمية إلى أجل غير مسمى.

Vim / Neovim: محرر نصوص يعمل بالكامل من سطر الأوامر. منحنى تعلمه حاد لكنه يوفر إنتاجية هائلة بعد إتقانه. شائع جداً بين مطوري Linux.

IDLE: بيئة التطوير المدمجة مع بايثون. تأتي تلقائياً مع تثبيت بايثون. بسيطة جداً ومناسبة للمبتدئين لتجربة الأكواد القصيرة.

Thonny: IDE بسيط موجه للمبتدئين تماماً. يوضح كيفية تنفيذ الكود خطوة بخطوة بشكل مرئي. ممتاز لتعلم أساسيات البرمجة.

خلاصة الملحق ب

  • VS Code هو الخيار الأفضل للمبتدئين وللاستخدام اليومي.
  • PyCharm مثالي للمشاريع الكبيرة والاحترافية.
  • جرب عدة محررات واختر ما يناسب أسلوب عملك.
  • بغض النظر عن المحرر الذي تختاره، الأهم هو فهم أساسيات لغة بايثون نفسها.

الملحق ج: الحصول على المساعدة

كل مبرمج يحتاج إلى المساعدة أحياناً، وهذا أمر طبيعي تماماً. في هذا الملحق، سنستعرض أفضل المصادر والطرق للحصول على المساعدة عندما تعلق في مشكلة برمجية.

أولاً: حاول أن تحل المشكلة بنفسك

قبل أن تطلب المساعدة من الآخرين، جرب هذه الخطوات بنفسك:

  • اقرأ رسالة الخطأ بعناية: بايثون تخبرك بالضبط أين وقع الخطأ ونوعه. ابحث عن اسم الخطأ (مثل SyntaxError أو TypeError) ورقم السطر.
  • استخدم print() للتحقق: أضف جمل طباعة مؤقتة لترى قيم المتغيرات في نقاط مختلفة من البرنامج.
  • ابحث في جوجل: اكتب رسالة الخطأ أو وصف المشكلة في جوجل بالعربية أو الإنجليزية.
  • أعد قراءة الكود: خذ استراحة قصيرة ثم اقرأ الكود مرة أخرى. أحياناً يختفي الخطأ بمجرد إلقاء نظرة جديدة.

وثائق بايثون الرسمية

المرجع الرسمي للغة بايثون هو أفضل مصدر للمعلومات الدقيقة والموثوقة.

  • الموقع الرسمي: docs.python.org/3/ - يحتوي على كل شيء: دروس تعليمية، مرجع المكتبة القياسية، ودليل تثبيت.
  • Python Tutor: موقع تفاعلي يشرح كيفية تنفيذ كود بايثون خطوة بخطوة بشكل مرئي على pythontutor.com.
  • PEP 8: دليل أسلوب كتابة كود بايثون. مفيد جداً لتعلم كتابة كود نظيف وقابل للقراءة.

مجتمعات بايثون العربية والعالمية

المجتمعات العربية:

  • مجموعات فيسبوك: مجموعات مثل "بايثون بالعربي" و"مبرمجي بايثون العرب" تقدم دعماً ممتازاً باللغة العربية.
  • منصة أكاديمية الزيرو: محتوى عربي متميز في تعليم البرمجة بلغة بايثون.
  • تليجرام: قنوات ومجموعات بايثون عربية متعددة.
  • منصة طاسك: محتوى عربي متخصص في البرمجة وتعلم الآلة.

المجتمعات العالمية:

  • Stack Overflow: أكبر موقع للأسئلة والأجوبة البرمجية في العالم. ابحث عن مشكلتك أولاً، وإذا لم تجد حلاً، اطرح سؤالاً واضحاً مع تضمين الكود ورسالة الخطأ.
  • Reddit: مجتمع r/learnpython مخصص للمبتدئين في بايثون. مجتمع ودود ومفيد جداً.
  • Python.org Community: قوائم بريدية ومنتديات رسمية لمستخدمي بايثون.
  • Discord / Slack: خوادم متخصصة في بايثون حيث يمكنك الحصول على مساعدة فورية.

كيف تطرح سؤالاً برمجياً بشكل صحيح

طرح سؤال جيد هو مهارة بحد ذاتها. اتبع هذه الإرشادات للحصول على إجابات مفيدة:

  1. ابحث أولاً: تأكد من أن سؤالك لم يُطرح ويُجاب عنه من قبل.
  2. كن محدداً: بدلاً من "الكود لا يعمل"، قل: "عند تشغيل الحلقة for، تظهر رسالة IndexError: list index out of range".
  3. ضَمّن الكود ورسالة الخطأ: انسخ الكود الذي يسبب المشكلة ورسالة الخطأ الكاملة.
  4. اشرح ما جربته: أخبر الآخرين بالخطوات التي قمت بها لحل المشكلة.
  5. استخدم Minimal Reproducible Example: اختصر الكود إلى أصغر جزء يسبب المشكلة.
  6. التنسيق مهم: استخدم تنسيق الكود في المنتديات ليسهل قراءة الأكواد.
نصيحة ذهبية: اقرأ مساعدة بايثون المدمجة! اكتب help(اسم_الدالة) في الوضع التفاعلي للحصول على شرح لأي دالة أو وحدة. مثال: help(print) أو help(list.append).

أدوات المساعدة المدمجة في بايثون

بايثون توفر أدوات مساعدة قوية مدمجة في اللغة نفسها:

  • help(): شرح تفاعلي لأي دالة أو وحدة.
  • dir(): عرض جميع الخصائص والدوال المتاحة لكائن معين.
  • type(): معرفة نوع أي قيمة أو متغير.
  • __doc__: عرض سلسلة التوثيق (docstring) لأي دالة.

خلاصة الملحق ج

  • حاول حل المشكلة بنفسك أولاً بقراءة رسالة الخطأ واستخدام print().
  • استخدم docs.python.org كمرجع أساسي.
  • انضم إلى مجتمعات بايثون العربية والعالمية للاستفادة من خبرات الآخرين.
  • اطرح أسئلتك بشكل واضح ومنظم مع تضمين الكود ورسالة الخطأ.
  • تذكر قول اريك ماتيس: "كل مبرمج محترف كان مبتدئاً في يوم من الأيام. لا تتردد في طلب المساعدة."

الملحق د: استخدام Git للتحكم بالإصدارات

Git هو نظام التحكم بالإصدارات (Version Control System) الأكثر استخداماً في عالم البرمجة. يتيح لك تتبع التغييرات في الكود الخاص بك، والعودة إلى إصدارات سابقة، والتعاون مع الآخرين على نفس المشروع. في هذا الملحق، سنتعلم أساسيات Git.

ما هو التحكم بالإصدارات ولماذا هو مهم؟

تخيل أنك تعمل على مشروع بايثون وتريد تجربة فكرة جديدة. ماذا لو أفسدت الكود؟ بدون Git، ستضطر إلى نسخ الملفات يدوياً قبل كل تعديل (مثل project_final_v3_actually_final.py). Git يحل هذه المشكلة تماماً.

فوائد Git:

  • تتبع كل تغيير تجريه على الكود مع إمكانية الرجوع لأي إصدار سابق.
  • العمل على فروع (Branches) مختلفة لتجربة أفكار جديدة دون التأثير على الكود الرئيسي.
  • التعاون مع فريق بسلاسة عبر دمج التغييرات.
  • نسخ احتياطي للمشروع على خدمات مثل GitHub و GitLab.
  • توثيق تطور المشروع عبر رسائل الالتزام (Commit Messages).

تثبيت Git

Windows:

  1. اذهب إلى git-scm.com وحمل المثبت.
  2. شغّل المثبت. يمكنك ترك الخيارات الافتراضية في معظم الخطوات.
  3. اختر "Git from the command line and also from 3rd-party software" عند ظهور الخيار.
  4. بعد التثبيت، افتح Git Bash أو موجه الأوامر واكتب git --version للتأكيد.

macOS:

# باستخدام Homebrew brew install git # أو التثبيت عبر Xcode Command Line Tools xcode-select --install

Linux (Ubuntu/Debian):

sudo apt update sudo apt install git

إعداد Git لأول مرة

قبل البدء، قم بتعريف اسمك وبريدك الإلكتروني. هذه المعلومات ستظهر في كل التزام (Commit) تقوم به:

git config --global user.name "اسمك" git config --global user.email "بريدك@example.com"

للتحقق من الإعدادات:

git config --list

أوامر Git الأساسية

سنستعرض الآن أهم أوامر Git التي ستحتاجها يومياً. لننشئ مشروعاً تجريبياً لتطبيق ما نتعلمه:

1. إنشاء مستودع (Repository):

# أنشئ مجلداً جديداً وادخل إليه mkdir my_python_project cd my_python_project # أنشئ مستودع Git جديد git init

2. إنشاء ملف وإضافة محتوى:

# أنشئ ملف hello.py echo 'print("Hello, Git!")' > hello.py # تحقق من حالة المستودع git status

3. إضافة الملفات إلى منطقة التجهيز (Staging Area):

# أضف ملفاً واحداً git add hello.py # أو أضف جميع الملفات git add .

4. الالتزام بالتغييرات (Commit):

git commit -m "إضافة ملف hello.py الأول"
نصيحة: اكتب رسائل commit واضحة ومفيدة. تخيل أنك تقرأ هذه الرسالة بعد 6 أشهر، هل ستفهم ما الذي تغير؟ استخدم العربية أو الإنجليزية، المهم أن تكون واضحة.

5. عرض تاريخ الالتزامات:

git log # عرض بشكل مختصر git log --oneline

6. مقارنة التغييرات:

# التغييرات غير المضافة git diff # الفرق بين آخر commit والملفات الحالية git diff HEAD

الفروع (Branches)

الفروع تسمح لك بالعمل على ميزات جديدة بشكل منعزل عن الكود الرئيسي. الفرع الرئيسي يسمى main (أو master في الإصدارات القديمة).

# اعرض كل الفروع git branch # أنشئ فرعاً جديداً git branch new-feature # انتقل إلى فرع آخر git checkout new-feature # أنشئ وانتقل إلى فرع جديد مباشرة (اختصار للخطوتين السابقتين) git checkout -b new-feature

بعد إجراء تغييرات في الفرع الجديد، يمكنك دمجها مع الفرع الرئيسي:

# عد إلى الفرع الرئيسي git checkout main # ادمج الفرع الجديد مع الرئيسي git merge new-feature

GitHub والتواصل مع المستودعات البعيدة

GitHub هي خدمة استضافة لمستودعات Git على الإنترنت. تتيح لك نسخ مشروعك احتياطياً ومشاركته مع الآخرين.

ربط مشروعك المحلي بـ GitHub:

  1. أنشئ حساباً على github.com.
  2. اضغط على زر "+" في أعلى اليمين ثم "New repository".
  3. سمِّ المستودع (مثلاً my-python-project) واترك الخيارات الافتراضية.
  4. في صفحة المستودع الجديد، ستجد أوامر لربطه بمشروعك المحلي. الأهم هو:
    git remote add origin https://github.com/اسمك/my-python-project.git git push -u origin main

الأوامر الأساسية للتعامل مع المستودع البعيد:

# رفع التغييرات المحلية إلى GitHub git push # جلب آخر التغييرات من GitHub git pull # نسخ (Clone) مستودع من GitHub إلى جهازك git clone https://github.com/اسم_المستخدم/اسم_المشروع.git

ممارسات مهمة في Git

  • قم بالـ commit بانتظام: بعد كل خطوة صغيرة ناجحة، قم بعمل commit. هذا يجعل العودة إلى إصدار سابق أسهل.
  • اكتب رسائل commit وصفية: "تم إصلاح خطأ القسمة على صفر" أفضل من "تحديث".
  • استخدم .gitignore: أنشئ ملف .gitignore في مجلد المشروع لتجاهل الملفات التي لا تريد تتبعها (مثل __pycache__/ و .env و venv/).
  • لا ترفع الملفات الكبيرة أو الحساسة: لا تضع كلمات السر أو المفاتيح الخاصة في المستودع.
  • اسحب التغييرات (pull) قبل الدفع (push): دائماً اسحب أحدث التغييرات من المستودع البعيد قبل رفع تغييراتك لتجنب التعارضات.
# نموذج ملف .gitignore لمشروع بايثون __pycache__/ *.py[cod] *.so .env venv/ .vscode/ .idea/ *.db
تحذير: إذا تعارضت التغييرات (Merge Conflict)، لا تقلق! هذا أمر طبيعي. اقرأ رسالة الخطأ، افتح الملف، وستجد أقساماً محددة بعلامات <<<<<<< و ======= و >>>>>>>. اختر التغيير الصحيح واحذف العلامات، ثم قم بـ commit مرة أخرى.

خلاصة الملحق د

  • Git يتتبع تغييرات الكود ويتيح لك الرجوع لأي إصدار سابق.
  • الأوامر الأساسية: git init، git add، git commit، git push، git pull.
  • الفروع تسمح بالعمل على ميزات متعددة بشكل مستقل.
  • GitHub يخزن مستودعاتك على الإنترنت ويتيح التعاون مع الآخرين.
  • استخدم .gitignore واكتب رسائل commit واضحة.
  • التعارضات طبيعية وقابلة للحل بسهولة.
ختاماً: هذه الملاحق صممت لمساعدتك على البدء. كلما تقدمت في رحلتك مع بايثون، ستجد أن هذه الأدوات تصبح جزءاً طبيعياً من سير عملك. تذكر أن التعلم المستمر والممارسة هما مفتاح الإتقان. وفقك الله في رحلتك البرمجية!
