الفصل السابع: إدخال المستخدم وحلقات while

User Input and while Loops — التفاعل مع المستخدم والتحكم في تدفق البرنامج

← العودة إلى الرئيسية

معظم البرامج الحقيقية تحتاج إلى التفاعل مع المستخدم — قراءة مدخلاته، معالجتها، والاستجابة لها. في هذا الفصل، ستتعلم كيفية استخدام الدالة input() لجلب البيانات من المستخدم، وكيفية استخدام حلقات while لتكرار تنفيذ الأوامر حتى يتحقق شرط معين. هاتان الأداتان تشكلان أساس بناء برامج تفاعلية حقيقية.

كيف تعمل الدالة input()

الدالة input() توقف تنفيذ البرنامج مؤقتاً وتنتظر حتى يقوم المستخدم بإدخال نص ثم يضغط مفتاح Enter. بعد ذلك، تأخذ الدالة النص المُدخل وتُرجعه كقيمة نصية (string) يمكن تخزينها في متغير واستخدامها في البرنامج.

المثال التالي يطلب من المستخدم إدخال اسمه ثم يطبع رسالة ترحيب:

message = input("Tell me something, and I will repeat it back to you: ") print(message)
شرح الكود: يطلب هذا الكود من المستخدم إدخال نص عبر الدالة input()، حيث تظهر الرسالة "Tell me something..." كدليل إرشادي. بعد كتابة المستخدم لأي نص والضغط على Enter، يُخزَّن الإدخال في المتغير message. السطر الثاني يستخدم print() لإعادة طباعة ما كتبه المستخدم تماماً كما هو. المخرجات المتوقعة: إذا أدخل المستخدم "Hello"، سيظهر في الشاشة "Hello".

عند تشغيل هذا الكود، سيظهر النص داخل الأقواس كرسالة (prompt) في شاشة البرنامج، وسينتظر إدخال المستخدم. أي شيء يكتبه المستخدم يُخزَّن في المتغير message ثم يُطبع مرة أخرى.

كتابة رسائل توضيحية واضحة (Writing Clear Prompts)

يجب أن تكون الرسالة التي تمررها إلى input() واضحة ومحددة حتى يعرف المستخدم بالضبط ما هو المطلوب منه. يمكنك بناء رسالة متعددة الأسطر باستخدام متغير منفصل:

prompt = "If you tell us who you are, we can personalize the messages you see." prompt += "\nWhat is your first name? " name = input(prompt) print(f"\nHello, {name}!")
شرح الكود: نبدأ بتعريف متغير prompt يحمل الجملة الترحيبية الأولى. السطر الثاني يستخدم += لإضافة سطر جديد (\n) وسؤال عن الاسم إلى نفس المتغير، مما يُوسِّع النص الأصلي دون الحاجة لكتابته من جديد. بعدها نمرر prompt إلى input() لتظهر الرسالة كاملة للمستخدم. وأخيراً نطبع رسالة ترحيب مخصصة باستخدام f-string، حيث سيظهر الاسم الذي أدخله المستخدم مباشرة داخل النص — مثلاً "Hello, Ahmed!".

هنا استخدمنا عامل الإسناد المُركب += لإضافة سطر جديد إلى نهاية النص الأصلي. هذا يجعل الكود أنظف وأسهل للقراءة عندما تكون الرسالة طويلة.

استخدام int() لقبول المدخلات الرقمية

الدالة input() تُرجع دائماً نصاً (string)، حتى لو كتب المستخدم أرقاماً. إذا أردت استخدام القيمة المُدخلة في عمليات حسابية، يجب تحويلها إلى عدد باستخدام الدالة int():

height = input("How tall are you, in inches? ") height = int(height) if height >= 48: print("\nYou're tall enough to ride!") else: print("\nYou'll be able to ride when you're a little older.")
شرح الكود: يبدأ البرنامج بسؤال المستخدم عن طوله بالبوصات باستخدام input()، والتي تُرجِع دائماً قيمة نصية (string). السطر الثاني هو الأهم: height = int(height) يحوّل النص المُدخل إلى عدد صحيح حتى نتمكن من إجراء مقارنات رقمية. بعد ذلك، تتحقق جملة if من أن الطول أكبر من أو يساوي 48، وتُطبَع رسالة مناسبة حسب النتيجة. بدون التحويل بـ int()، كانت المقارنة height >= 48 ستفشل لأن بايثون لا تقارن نصاً بعدد.

في هذا المثال، يقوم البرنامج بتحويل القيمة النصية إلى عدد صحيح ثم يقارنها مع الحد الأدنى للطول المسموح به لدخول اللعبة. بدون استخدام int()، كانت المقارنة ستفشل لأننا لا يمكننا مقارنة نص بعدد.

عامل الموديولو (المعامل) — The Modulo Operator

عامل الموديولو (%) هو عملية حسابية تُرجع باقي القسمة. يُستخدم بكثرة في البرمجة لتحديد ما إذا كان عدد ما زوجياً أو فردياً، أو للتحقق من قابلية القسمة.

# باقي قسمة 4 على 3 = 1 print(4 % 3) # باقي قسمة 5 على 3 = 2 print(5 % 3) # باقي قسمة 6 على 3 = 0 (يقبل القسمة) print(6 % 3) # باقي قسمة 7 على 3 = 1 print(7 % 3)
شرح الكود: يُظهر هذا الكود عامل الموديولو % الذي يُرجِع باقي القسمة بين عددين. في السطر الأول، 4 % 3 يُرجِع 1 لأن 4 ÷ 3 = 1 والباقي 1. السطر الثاني 5 % 3 يُرجِع 2. السطر الثالث 6 % 3 يُرجِع 0 لأن 6 تقبل القسمة على 3 بدون باقٍ. السطر الرابع 7 % 3 يُرجِع 1. المخرجات ستكون: 1, 2, 0, 1 — كل في سطر منفصل. هذا العامل أساسي لتحديد الأعداد الزوجية والفردية.

إذا كان باقي قسمة عدد على 2 يساوي 0، فهذا يعني أن العدد زوجي (even). أما إذا كان الباقي 1، فالعدد فردي (odd).

number = input("Enter a number, and I'll tell you if it's even or odd: ") number = int(number) if number % 2 == 0: print(f"\nThe number {number} is even.") else: print(f"\nThe number {number} is odd.")
شرح الكود: يطلب البرنامج من المستخدم إدخال رقم، ثم يُحوّله من نص إلى عدد صحيح باستخدام int(). جملة if تتحقق مما إذا كان باقي قسمة الرقم على 2 يساوي 0 — إذا كان الجواب نعم فالرقم زوجي (even)، وإلا فهو فردي (odd). تستخدم جملة print تقنية f-string لإدراج الرقم الأصلي داخل رسالة الإخراج. المخرجات المتوقعة: إذا أدخل المستخدم 7، ستظهر رسالة "The number 7 is odd.".

استقبال الإدخال في بايثون 2.7

في بايثون 2.7 (وهي نسخة قديمة ومتقادمة)، كانت الدالة المستخدمة لقراءة الإدخال النصي هي raw_input(). أما الدالة input() في بايثون 2.7 فكانت تحاول تقييم (evaluate) الإدخال ككود بايثون، مما كان يُسبب مشاكل أمنية.

ملاحظة: بايثون 2.7 لم تعد مدعومة منذ يناير 2020. يُنصح بشدة باستخدام بايثون 3 في جميع المشاريع الجديدة. كل الأمثلة في هذا الكتاب تستخدم بايثون 3 والدالة input().

مقارنة بين الطريقتين:

# Python 3 — باستخدام input() name = input("What's your name? ") # Python 2.7 — باستخدام raw_input() name = raw_input("What's your name? ")
شرح الكود: يُظهر هذا الكود الفرق الجوهري بين بايثون 3 وبايثون 2.7 في قراءة المدخلات من المستخدم. في بايثون 3 (السطر الأول)، تُستخدم input() التي تُرجِع النص كما هو دون أي تعديل. في بايثون 2.7 (السطر الثاني)، كانت تُستخدم raw_input() لنفس الغرض، بينما input() في 2.7 كانت تحاول تنفيذ النص المُدخل ككود بايثون — مما يُشكّل خطراً أمنياً كبيراً. لهذا السبب، من الضروري استخدام بايثون 3 في جميع المشاريع الجديدة.

مقدمة عن حلقات while

حلقة for تأخذ مجموعة من العناصر وتنفذ كتلة من الكود مرة واحدة لكل عنصر. أما حلقة while فتُنفذ كتلة من الكود طالما شرط معين يتحقق (True). تتوقف الحلقة فقط عندما يُصبح الشرط False.

حلقة while في العمل

هذا مثال بسيط يحسب الأعداد من 1 إلى 5:

current_number = 1 while current_number <= 5: print(current_number) current_number += 1
شرح الكود: يبدأ المتغير current_number بالقيمة 1. تتحقق حلقة while من الشرط current_number <= 5 — طالما هذا الشرط صحيح (True)، يستمر التكرار. داخل الحلقة، يُطبَع الرقم أولاً ثم يزيد بمقدار 1 عبر current_number += 1. عندما يصل المتغير إلى 6 يصبح الشرط False وتتوقف الحلقة تلقائياً. المخرجات ستكون: 1, 2, 3, 4, 5 — كل رقم في سطر منفصل.

المتغير current_number يبدأ بالقيمة 1. حلقة while تفحص ما إذا كانت قيمة المتغير أقل من أو تساوي 5. طالما هذا الشرط صحيح، تطبع الحلقة الرقم ثم تزيده بمقدار 1. عندما يصل المتغير إلى 6، يُصبح الشرط False وتتوقف الحلقة.

السماح للمستخدم بتحديد وقت الخروج

يمكنك استخدام حلقة while لتشغيل البرنامج حتى يُدخل المستخدم كلمة مفتاحية للخروج، مثل 'quit':

prompt = "\nTell me something, and I will repeat it back to you:" prompt += "\nEnter 'quit' to end the program. " message = "" while message != 'quit': message = input(prompt) if message != 'quit': print(message)
شرح الكود: نبدأ بتعريف رسالة prompt من سطرين باستخدام += ونضبط message = "" كنص فارغ لتجنب خطأ المقارنة في السطر التالي. تفحص الحلقة شرط message != 'quit' — طالما أن المستخدم لم يكتب "quit"، يستمر التكرار. داخل الحلقة، نأخذ إدخال المستخدم عبر input()، وإذا كان الإدخال مختلفاً عن "quit" نطبعه. عند كتابة "quit"، يصبح الشرط False وتنتهي الحلقة دون طباعة كلمة quit نفسها.

هنا نضع message = "" في البداية حتى تتمكن بايثون من تنفيذ جملة while (لأنها تحتاج لمتغير للمقارنة). داخل الحلقة، نأخذ إدخال المستخدم، وإذا لم يكن 'quit' نطبعه. إذا كتب 'quit'، ينتهي الشرط وتتوقف الحلقة.

استخدام راية (Flag)

في البرامج الأكثر تعقيداً، قد تحتاج عدة شروط مختلفة لتحديد متى يجب أن تتوقف الحلقة. بدلاً من كتابة شرط طويل ومعقد في جملة while، يمكنك تعريف متغير منطقي (boolean) يُسمى "flag" يشير إلى ما إذا كان البرنامج يجب أن يستمر في العمل أم لا:

prompt = "\nTell me something, and I will repeat it back to you:" prompt += "\nEnter 'quit' to end the program. " active = True while active: message = input(prompt) if message == 'quit': active = False else: print(message)
شرح الكود: هذا المثال يُعادل السابق لكن باستخدام راية (flag) — متغير منطقي يُسمى active. نضبط active = True وتستمر الحلقة while active: طالما الراية True. داخل الحلقة، إذا كتب المستخدم "quit" نغير الراية إلى False فتنتهي الحلقة فوراً؛ وإلا نطبع الرسالة. هذا الأسلوب أفضل للبرامج المعقدة حيث يمكن إضافة شروط إضافية تغير الراية، مما يجعل الكود أنظف وأسهل في الصيانة والتطوير.

نعطي الراية active القيمة True في البداية، وتستمر الحلقة طالما active تساوي True. داخل الحلقة، إذا كتب المستخدم 'quit' نغير قيمة الراية إلى False فتنتهي الحلقة. هذا الأسلوب يجعل الكود أكثر وضوحاً ويسهل إضافة شروط إضافية لاحقاً.

استخدام break للخروج من الحلقة

يمكنك استخدام الكلمة المحجوزة break للخروج فوراً من حلقة while أو for دون الحاجة إلى التحقق من شرط الحلقة:

prompt = "\nPlease enter the name of a city you have visited:" prompt += "\n(Enter 'quit' when you are finished.) " while True: city = input(prompt) if city == 'quit': break else: print(f"I'd love to go to {city.title()}!")
شرح الكود: هنا نستخدم while True: لإنشاء حلقة لا نهائية تنتهي فقط عند تنفيذ break. داخل الحلقة، نطلب من المستخدم إدخال اسم مدينة. إذا كتب "quit" يُنفَّذ break ليخرج من الحلقة فوراً إلى السطر الذي يليها. وإلا، تُطبَع رسالة تحتوي على اسم المدينة مع تنسيق title() لجعل الحرف الأول كبيراً. هذا النمط — "while True مع break" — شائع جداً في التطبيقات العملية لأنه يُبقي شرط الخروج داخل الحلقة وليس في توقيع while.

هنا نستخدم while True لتشغيل الحلقة إلى أجل غير مسمى، ونتوقف فقط عند تنفيذ break. هذه طريقة شائعة جداً في البرامج العملية.

ملاحظة: يمكنك استخدام break في أي حلقة بايثون، سواء كانت for أو while. عند تنفيذ break، ينتقل تدفق البرنامج فوراً إلى الكود الموجود بعد الحلقة مباشرة.

استخدام continue في الحلقة

بدلاً من الخروج الكامل من الحلقة، يمكنك استخدام continue لتخطي بقية الكود في التكرار الحالي والانتقال فوراً إلى التكرار التالي:

current_number = 0 while current_number < 10: current_number += 1 if current_number % 2 == 0: continue print(current_number)
شرح الكود: يبدأ الكود بعدّاد current_number قيمته 0، ثم يدخل حلقة while تستمر طالما العدد أقل من 10. في كل دورة، نزيد العدد أولاً بمقدار 1 ثم نتحقق من أنه زوجي باستخدام % 2 == 0. إذا كان زوجياً، ننفذ continue الذي يقفز فوراً إلى التكرار التالي متجاوزاً جملة print. المخرجات ستكون الأعداد الفردية فقط: 1, 3, 5, 7, 9 — كل عدد في سطر منفصل.

هذا الكود يطبع الأعداد الفردية فقط من 1 إلى 9. عندما يكون العدد زوجياً (باقي قسمته على 2 يساوي 0)، ننفذ continue الذي يتخطى جملة print وينتقل إلى التكرار التالي.

تجنب الحلقات اللانهائية

كل حلقة while تحتاج إلى طريقة للتوقف. إذا نسيتَ تحديث المتغير الذي يتحقق الشرط منه، ستصبح الحلقة لا نهائية (infinite loop) ويتجمد البرنامج أو يستمر في العمل إلى أجل غير مسمى:

# هذه حلقة لا نهائية — لا تفعل هذا! x = 1 while x <= 5: print(x) # نسينا إضافة x += 1
شرح الكود: يُظهر هذا الكود خطأً شائعاً جداً: حلقة لا نهائية (infinite loop). المتغير x يبدأ بـ 1 وتتكرر الحلقة طالما x <= 5. ولكن داخل الحلقة، نطبع x فقط بدون تحديث قيمته — أي لا يوجد x += 1. نتيجة لذلك، سيظل x يساوي 1 إلى الأبد ويستمر البرنامج في طباعة 1 بلا توقف. هذا يوضح أهمية التأكد من أن شرط الخروج سيصبح False في وقت ما داخل كل حلقة while.
تحذير: الحلقات اللانهائية خطأ شائع جداً بين المبتدئين. إذا توقف برنامجك عن الاستجابة أو استمر في الطباعة بلا توقف، اضغط Ctrl+C في الطرفية لإنهاء البرنامج قسراً. تأكد دائماً من أن شرط الخروج سيصبح False في وقت ما.

إذا كتبت برنامجاً به حلقة لا نهائية عن طريق الخطأ، فقط اضغط Ctrl+C وأصلح الكود.

استخدام حلقة while مع القوائم والقواميس

حتى الآن، تعاملنا مع حلقات while في اختبارات شرطية بسيطة. لكن القوة الحقيقية لهذه الحلقات تظهر عند استخدامها لتعديل القوائم والقواميس أثناء تشغيل البرنامج. حلقات for لا تسمح بتعديل القائمة التي يتم التكرار عليها، بينما حلقات while تمنحك هذه المرونة.

نقل العناصر من قائمة إلى أخرى

لنفترض أن لدينا قائمة بمستخدمين جدد لم يتم التحقق منهم بعد، ونريد نقلهم إلى قائمة المستخدمين الموثَّقين بعد التحقق منهم:

# ابدأ بمستخدمين لم يتم التحقق منهم unconfirmed_users = ['alice', 'brian', 'candace'] confirmed_users = [] # تحقق من كل مستخدم حتى لا يتبقى أي مستخدم غير موثق while unconfirmed_users: current_user = unconfirmed_users.pop() print(f"Verifying user: {current_user.title()}") confirmed_users.append(current_user) # عرض جميع المستخدمين الموثقين print("\nThe following users have been confirmed:") for confirmed_user in confirmed_users: print(confirmed_user.title())
شرح الكود: لدينا قائمتان: unconfirmed_users تضم مستخدمين جدد، وconfirmed_users فارغة في البداية. تستخدم الحلقة while unconfirmed_users: — طالما القائمة الأولى غير فارغة (شرط ضمني True)، نستخدم pop() لاستخراج آخر عنصر منها ونخزنه في current_user. نطبع رسالة تحقق ونضيف المستخدم إلى قائمة الموثقين عبر append(). بعد انتهاء الحلقة، نطبع جميع المستخدمين الموثقين باستخدام حلقة for. المخرجات ستظهر الأسماء بالترتيب: Candace, Brian, Alice — لأن pop() تأخذ من النهاية.

هنا نستخدم while unconfirmed_users — طالما أن القائمة ليست فارغة (شرط True)، نأخذ آخر عنصر منها باستخدام pop() ونضيفه إلى القائمة الثانية بعد طباعة رسالة التحقق. عندما تصبح القائمة الأولى فارغة، يتوقف الشرط وتنتهي الحلقة.

إزالة كل تكرارات قيمة محددة من قائمة

الدالة remove() تزيل التكرار الأول فقط من القيمة. إذا أردت إزالة كل التكرارات، تستخدم حلقة while:

pets = ['dog', 'cat', 'dog', 'goldfish', 'cat', 'rabbit', 'cat'] print(pets) while 'cat' in pets: pets.remove('cat') print(pets)
شرح الكود: القائمة pets تحتوي على 7 عناصر مع تكرار "cat" ثلاث مرات و"dog" مرتين. نطبع القائمة الأصلية أولاً، ثم نستخدم while 'cat' in pets: — تستمر الحلقة طالما أن القيمة "cat" لا تزال موجودة في القائمة. في كل تكرار، تزيل remove() التكرار الأول فقط لـ "cat". تتوقف الحلقة تلقائياً بعد إزالة جميع التكرارات. المخرجات ستظهر القائمة قبل وبعد: القائمة النهائية ['dog', 'dog', 'goldfish', 'rabbit'] خالية تماماً من "cat".

الحلقة تستمر طالما أن القيمة 'cat' موجودة في القائمة. في كل تكرار، تزيل remove() التكرار الأول لـ 'cat'. تتوقف الحلقة تلقائياً بعد إزالة جميع التكرارات.

ملء قاموس ببيانات المستخدم

يمكنك استخدام حلقة while لجمع بيانات من المستخدمين وتخزينها في قاموس. هذا مفيد جداً في بناء تطبيقات الاستبيانات:

responses = {} # تعيين راية للإشارة إلى استمرار الاستطلاع polling_active = True while polling_active: # طلب الاسم والإجابة من المستخدم name = input("\nWhat is your name? ") response = input("Which mountain would you like to climb someday? ") # تخزين الإجابة في القاموس responses[name] = response # معرفة ما إذا كان هناك مستخدم آخر يريد المشاركة repeat = input("Would you like to let another person respond? (yes/no) ") if repeat == 'no': polling_active = False # عرض نتائج الاستطلاع print("\n--- Poll Results ---") for name, response in responses.items(): print(f"{name} would like to climb {response}.")
شرح الكود: هذا برنامج استطلاع متكامل. ننشئ قاموساً فارغاً responses ونضبط راية polling_active = True. داخل حلقة while، نطلب اسم المستخدم وإجابته عن الجبل الذي يريد تسلقه، ونخزنهما في القاموس عبر responses[name] = response. ثم نسأل إذا كان هناك شخص آخر يريد المشاركة؛ إذا كانت الإجابة "no" نغير الراية إلى False. بعد الخروج من الحلقة، نطبع جميع النتائج باستخدام responses.items() التي تُرجع أزواج المفتاح-القيمة. المخرجات ستظهر اسم كل شخص والجبل الذي اختاره.

في هذا المثال، نستخدم راية polling_active للتحكم في الحلقة. في كل تكرار، نطلب اسم المستخدم وإجابته و نخزنهما في القاموس responses. ثم نسأل المستخدم إذا كان هناك شخص آخر يريد المشاركة. إذا كانت الإجابة 'no'، نغير الراية إلى False وتتوقف الحلقة. بعد ذلك، نعرض جميع الإجابات المخزنة في القاموس.

هذه هي الطريقة التي تعمل بها البرامج الحقيقية التي تجمع بيانات من المستخدمين — تستخدم حلقة while مع راية وتخزّن النتائج في هيكل بيانات (عادةً قائمة أو قاموس) لمعالجتها لاحقاً.

تمارين الفصل السابع

تمرين 7-1: تأجير السيارات
اكتب برنامجاً يسأل المستخدم عن نوع السيارة التي يريد استئجارها، ثم اطبع رسالة مثل "Let me see if I can find you a Subaru."
تمرين 7-2: حجز مطعم
اكتب برنامجاً يسأل المستخدم عن عدد الأشخاص في مجموعته. إذا كان العدد أكبر من 8، اطبع رسالة تفيد بأنهم سيحتاجون إلى الانتظار. وإلا، اطبع رسالة تفيد بأن الطاولة جاهزة.
تمرين 7-3: مضاعفات العدد 10
اسأل المستخدم عن رقم، ثم أخبره إذا كان هذا الرقم من مضاعفات العدد 10 أم لا.
تمرين 7-4: إضافات البيتزا
اكتب حلقة تطلب من المستخدم إضافة إضافات إلى البيتزا. استمر في الطلب حتى يكتب 'quit'. كلما أضاف المستخدم إضافة، اطبع رسالة تؤكد إضافتها.
تمرين 7-5: تذاكر السينما
اكتب حلقة تسأل المستخدم عن عمره، ثم تحدد سعر التذكرة: أقل من 3 سنوات: مجاناً، من 3 إلى 12 سنة: 10 دولارات، أكبر من 12 سنة: 15 دولاراً.
تمرين 7-6: ثلاث مخارج
أعد كتابة التمرين 7-4 أو 7-5 باستخدام شرط while واحد فقط لاختبار حالة التوقف (بدلاً من استخدام break أو راية).
تمرين 7-7: اللانهاية
اكتب حلقة لا نهائية وقم بتشغيلها. (لتوقفها اضغط Ctrl+C أو أغلق النافذة.)
تمرين 7-8: محل السندويشات
أنشئ قائمة sandwich_orders و قائمة فارغة finished_sandwiches. استخدم حلقة لنقل السندويشات من الأولى إلى الثانية، واطبع رسالة لكل سندويش يتم تحضيره.
تمرين 7-9: لا باستارمي
باستخدام القائمة من التمرين 7-8، تأكد من أن طلب 'pastrami' يظهر فيها ثلاث مرات. أضف كوداً في بداية البرنامج لطباعة رسالة تفيد بنفاد الباستارمي، ثم استخدم حلقة while لإزالة كل تكرارات 'pastrami' من القائمة.
تمرين 7-10: إجازة الأحلام
اكتب برنامجاً يستطلع المستخدمين عن مكان إجازة أحلامهم. استخدم حلقة لجمع البيانات (يمكن للمستخدم إيقاف البرنامج عند الانتهاء). بعد جمع الإجابات، اطبع النتائج.

بروتوكول التكرار - كيف تعمل for خلف الكواليس

عندما تكتب for x in obj:، بايثون تستدعي iter(obj) لتحصل على مُكرّر (Iterator)، ثم تستدعي next(iterator) في كل دورة حتى ترفع StopIteration. فهم هذا يفتح لك إمكانيات كبيرة:

# ما يحدث خلف الكواليس عند كتابة for x in [1, 2, 3]: it = iter([1, 2, 3]) # list_iterator while True: try: x = next(it) print(x) # 1, 2, 3 except StopIteration: break # أي كائن يعرّف __iter__ أو __getitem__ يمكن التكرار عليه
شرح الكود: يكشف هذا الكود ما يحدث خلف الكواليس عند استخدام حلقة for. تستدعي بايثون iter() على القائمة لتحصل على مُكرّر (iterator)، ثم تستدعي next() في كل دورة. كل استدعاء لـ next() يُرجِع العنصر التالي — 1, ثم 2, ثم 3. عندما تنفد العناصر، يرفع المُكرّر استثناء StopIteration الذي نلتقطه في except ونتوقف باستخدام break. المخرجات ستكون: 1, 2, 3 — تماماً كما تفعل حلقة for العادية.
# iter(callable, sentinel) — تكرار حتى قيمة محددة import random # يولد أرقاماً حتى تظهر القيمة 0 for val in iter(lambda: random.randint(0, 5), 0): print(val) # يطبع الأرقام حتى (وليس بما فيها) 0 # قراءة سطور ملف حتى سطر فارغ with open('data.txt') as f: for line in iter(f.readline, ''): print(line.strip())
شرح الكود: هذا نمط متقدم من بايثون — الدالة iter() تأخذ وسيطتين: دالة تُستدعى في كل تكرار، وقيمة حارسة (sentinel) يتوقف عندها التكرار. في المثال الأول، lambda: random.randint(0, 5) تُولّد أرقاماً عشوائية بين 0 و5، وتتوقف الحلقة عندما يظهر الرقم 0 (الرقم 0 نفسه لا يُطبَع). المثال الثاني يوضح قراءة سطور ملف حتى الوصول إلى سطر فارغ، وهو نمط مفيد جداً لقراءة الملفات ذات التنسيق المحدد.
# إنشاء مكرّر خاص — تنفيذ بروتوكول التكرار class CountDown: def __init__(self, start): self.n = start def __iter__(self): return self def __next__(self): if self.n < 0: raise StopIteration val = self.n self.n -= 1 return val for i in CountDown(3): print(i) # 3, 2, 1, 0
شرح الكود: نعرّف كلاس CountDown يطبّق بروتوكول التكرار عبر تعريف __iter__ (الذي يُرجِع self) و __next__. عند بدء التكرار، self.n يساوي الرقم الذي بدأنا به (3). كل استدعاء لـ next() يُرجِع القيمة الحالية ثم يُنقِصها بمقدار 1. عندما يصبح self.n أقل من 0، نرفع استثناء StopIteration لإنهاء التكرار. المخرجات ستكون: 3, 2, 1, 0 — عدّ تنازلياً من 3 إلى 0. هذا يوضح كيفية إنشاء كائنات مخصصة قابلة للتكرار في بايثون.
💡 من Fluent Python: بايثون تستخدم التكرار الكسول (Lazy) — لا تولّد القيم إلا عند الطلب. iter(callable, sentinel) مثالي لقراءة الملفات وأخذ القراءات من sensors. الكائنات التي تنفذ __getitem__ (بدون __iter__) لا تزال قابلة للتكرار لكن بكفاءة أقل.

  • كيف تتجنب الحلقات اللانهائية وكيف توقفها باستخدام Ctrl+C
  • كيف تستخدم حلقات while مع القوائم لنقل العناصر وإزالة التكرارات
  • كيف تستخدم حلقات while مع القواميس لجمع بيانات المستخدم
💡 نصيحة ذهبية: استخدم "sentinel pattern" مع while: متغير يتحكم في الحلقة، يتغير عند الإنهاء. مثلاً: active = True; while active: ... if condition: active = False. للتحقق من صحة الإدخال، استخدم while True: ... if valid: break — أنظف وأسهل للقراءة.
⚠️ خطأ شائع: الحلقات اللانهائية — تنسى تحديث المتغير الذي تختبره الشرط. إذا كتبت while x < 10: ولم تزد x داخل الحلقة، ستستمر للأبد. في بايثون، استخدم Ctrl+C لإيقافها. أيضاً، المقارنة != تختلف عن is not — الأول تقارن القيم، الثاني يقارن الكائنات.
