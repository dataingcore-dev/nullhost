🤖 أساسيات الذكاء الاصطناعي

NLP، Embeddings، Transformers، واستخدام APIs الذكاء الاصطناعي

← العودة إلى الرئيسية

هذا الفصل يغطي المفاهيم الأساسية للذكاء الاصطناعي الحديث — من معالجة اللغة الطبيعية (NLP) إلى النماذج الكبيرة (LLMs) و RAG (Retrieval-Augmented Generation).

1. NLP — معالجة اللغة الطبيعية

NLP (Natural Language Processing) هو فرع الذكاء الاصطناعي الذي يتعامل مع فهم وتوليد اللغة البشرية. الخطوة الأولى في أي نموذج NLP هي الـ Tokenization — تحويل النص إلى أرقام يمكن للنموذج فهمها. كل كلمة أو جزء منها يصبح token، وكل token له رقم معرف (ID) فريد في قاموس النموذج.

# تثبيت المكتبات pip install transformers torch sentence-transformers # Tokenization — تحويل النص إلى أرقام from transformers import AutoTokenizer tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased") tokens = tokenizer("Hello, how are you?") print(tokens) # {'input_ids': [101, 7592, 1010, 2129, 2024, 2017, 1029, 102], # 'attention_mask': [1, 1, 1, 1, 1, 1, 1, 1]}
🔍 كيف يعمل؟ AutoTokenizer.from_pretrained تحمّل الـ tokenizer الخاص بنموذج BERT (مدرّب مسبقاً على نصوص إنجليزية). tokenizer() يحوّل الجملة إلى: input_ids (أرقام تمثل كل token — 101 = [CLS] بداية الجملة، 102 = [SEP] نهاية، 7592 = "Hello"، إلخ) و attention_mask (1 = انتبه لهذا token، 0 = تجاهله). النموذج يحتاج أرقاماً، ليس نصوصاً!

2. Embeddings — التمثيل العددي للنصوص

التضمينات (Embeddings) هي تحويل النصوص إلى متجهات رقمية (vectors) بأبعاد ثابتة — مثلاً 384 أو 768 أو 1536 بُعداً. الفكرة: النصوص المتشابهة دلالياً (في المعنى) يكون لها متجهات قريبة من بعضها في الفضاء الرياضي. هذا هو أساس البحث الدلالي (Semantic Search) والتجميع (Clustering).

from sentence_transformers import SentenceTransformer model = SentenceTransformer('all-MiniLM-L6-v2') # 384-dimension sentences = [ "أنا أحب البرمجة", "الذكاء الاصطناعي رائع", "أحب تعلم أشياء جديدة" ] embeddings = model.encode(sentences) print(embeddings.shape) # (3, 384) # حساب التشابه (Cosine Similarity) from sklearn.metrics.pairwise import cosine_similarity sim = cosine_similarity([embeddings[0]], [embeddings[1]]) print(sim) # 0.72 — تشابه عالي
🔍 كيف يعمل؟ SentenceTransformer('all-MiniLM-L6-v2') يحمّل نموذج embedding صغير وسريع (384 بُعداً). encode يحوّل كل جملة إلى متجه رقمي. embeddings.shape = (3, 384) — 3 جمل، كل منها متجه بـ 384 رقم. Cosine Similarity يحسب الزاوية بين متجهين: 1 = متطابقان، 0 = لا تشابه، -1 = متعاكسان. نتيجة 0.78 تعني أن الجملتين متشابهتان جداً في المعنى.

3. Transformers — نماذج المحولات

Transformers هي البنية المعمارية التي تقف وراء كل نماذج LLM الحديثة (GPT, BERT, LLaMA). تستخدم آلية "الانتباه" (Attention) لفهم العلاقات بين كل كلمة وكل كلمة أخرى في النص. مكتبة transformers من Hugging Face تجعل استخدام هذه النماذج سهلاً بخطوط قليلة.

from transformers import pipeline # تصنيف المشاعر classifier = pipeline("sentiment-analysis") result = classifier("I loved the new course, it was amazing!") print(result) # [{'label': 'POSITIVE', 'score': 0.999}] # توليد النص generator = pipeline("text-generation", model="gpt2") output = generator("The future of AI is", max_length=50) print(output[0]['generated_text'])
🔍 كيف يعمل؟ pipeline تُبسّط استخدام النماذج: تحمّل النموذج والـ tokenizer تلقائياً. sentiment-analysis يستخدم نموذجاً مدرّباً لتصنيف المشاعر (إيجابي/سلبي) — النتيجة label='POSITIVE' بثقة 99.9%. text-generation مع GPT2 يكمّل النص: يعطي "The future of AI is..." ويولّد متابعة بناءً على ملايين النصوص التي تدرّب عليها. max_length=50 يحدد أقصى عدد من tokens للمخرَج.

4. Vector Databases وقواعد البيانات المتجهة

قواعد البيانات المتجهة (Vector Databases) تسمح بتخزين embeddings والبحث فيها عن الأكثر تشابهاً. هذا هو أساس RAG وأنظمة التوصية. لكن يمكنك البدء ببساطة باستخدام numpy لحساب التشابه، كما يوضح المثال التالي:

# تخزين embeddings والبحث فيها from sentence_transformers import SentenceTransformer import numpy as np model = SentenceTransformer('all-MiniLM-L6-v2') docs = [ "FastAPI is a modern web framework for Python", "PostgreSQL is a relational database", "Docker containers are lightweight and portable" ] doc_embeddings = model.encode(docs) # سؤال — تحويله إلى embedding query = "What is FastAPI?" query_emb = model.encode([query]) # البحث عن الأكثر تشابهاً scores = cosine_similarity(query_emb, doc_embeddings) best_idx = np.argmax(scores) print(docs[best_idx]) # أقرب وثيقة للسؤال
🔍 كيف يعمل؟ أولاً نُحوّل كل وثيقة إلى embedding باستخدام النموذج. ثم نُحوّل سؤال المستخدم إلى embedding أيضاً. cosine_similarity يحسب التشابه بين السؤال وكل وثيقة — يُعيد مصفوفة من 3 أرقام (مثلاً [0.92, 0.31, 0.15]). np.argmax يُعيد Index أعلى تشابه (0) — أي "FastAPI is a modern web framework for Python". هذا هو البحث الدلالي الأساسي!
ملاحظة: قواعد البيانات المتجهة مثل ChromaDB و Qdrant و Pinecone تسمح بالبحث عن التشابه على نطاق واسع.

5. RAG — Retrieval-Augmented Generation

RAG هو نمط يجمع بين استرجاع المعلومات (Retrieval) وتوليد النص (Generation). الفكرة: بدلاً من سؤال النموذج مباشرة (الذي قد تكون معرفته محدودة أو قديمة)، نبحث أولاً في قاعدة معرفتنا عن الوثائق ذات الصلة، ثم نُمرّرها للنموذج كسياق. هذا يُحسّن دقة الإجابات ويُقلل الهلوسة (hallucination).

# RAG: استرجاع + توليد # الخطوات: 1. استرجع الوثائق ذات الصلة 2. أضفها للسياق 3. ولّد الإجابة from openai import OpenAI import chromadb client = OpenAI(api_key="sk-...") def rag_query(question: str) -> str: # 1. استرجاع الوثائق ذات الصلة results = vector_db.similarity_search(question, k=3) context = "\n".join([doc.page_content for doc in results]) # 2. إنشاء الـ prompt مع السياق prompt = f"أجب السؤال بناءً على السياق التالي:\n{context}\n\nسؤال: {question}" # 3. توليد الإجابة response = client.chat.completions.create( model="gpt-4", messages=[{"role": "user", "content": prompt}] ) return response.choices[0].message.content
🔍 كيف يعمل؟ (1) similarity_search يبحث في قاعدة المتجهات عن أقوى 3 وثائق للسؤال. (2) نضمّ الوثائق في نص واحد (context). (3) نُنشئ prompt يطلب من GPT-4 الإجابة بناءً على هذا السياق فقط — هذا يمنع النموذج من التخمين أو استخدام معرفته القديمة. RAG هو العمود الفقري لتطبيقات AI Chat الحديثة مثل ChatGPT مع البحث في الويب.

6. استخدام APIs نماذج LLM

أسهل طريقة لاستخدام نماذج LLM هي عبر APIs — لا تحتاج GPU ولا تثبيت نماذج ضخمة. أشهر المزوّدين: OpenAI, Hugging Face, Anthropic. كلها توفر واجهة مشابهة: إرسال رسائل (messages) واستقبال ردود.

# OpenAI API from openai import OpenAI client = OpenAI(api_key="sk-your-key") response = client.chat.completions.create( model="gpt-4", messages=[ {"role": "system", "content": "أنت مساعد خبير في البرمجة."}, {"role": "user", "content": "اشرح polymorphism في OOP"} ] ) print(response.choices[0].message.content)
🔍 كيف يعمل؟ نظام المراسلة (messages): system message يحدد شخصية المساعد ("أنت خبير برمجة")، user message هو سؤال المستخدم. model="gpt-4" يحدد النموذج — كل نموذج له قدرات وسعر مختلف. response.choices[0].message.content يحتوي على النص الذي ولّده النموذج. لا تشارك api_key أبداً — استخدم متغيرات البيئة: os.getenv("OPENAI_API_KEY").
# Hugging Face Inference API (مجاني) import requests API_URL = "https://api-inference.huggingface.co/models/gpt2" headers = {"Authorization": "Bearer hf_your_token"} data = {"inputs": "The future of AI"} response = requests.post(API_URL, headers=headers, json=data) print(response.json())
🔍 كيف يعمل؟ Hugging Face Inference API يتيح استخدام آلاف النماذج المجانية عبر HTTP. نرسل طلب POST إلى URL النموذج مع النص المُدخل. الـ API يُعيد النص المُولّد. المميزات: مجاني للاستخدام المحدود، يدعم مئات النماذج، لا يحتاج تثبيت. العيوب: أبطأ من OpenAI، محدود الاستخدام. مثالي للتجربة والنماذج الأولية.

7. AI Fundamentals — المفاهيم الأساسية

  • Tokenization: تحويل النص إلى tokens
  • Embeddings: تمثيل النص كمتجهات رقمية
  • Attention: آلية الانتباه — فهم العلاقات بين الكلمات
  • Transformers: بنية تعتمد على self-attention بالكامل
  • Fine-tuning: تدريب نموذج موجود على بيانات جديدة
  • RAG: دمج الاسترجاع مع التوليد للإجابات الدقيقة
