SQL - قواعد البيانات العلائقية

Structured Query Language — لغة الاستعلام البنيوية لقواعد البيانات

SQL (Structured Query Language) هي اللغة القياسية للتعامل مع قواعد البيانات العلائقية. تتيح لك إنشاء البيانات وتعديلها واستعلامها بسهولة. في هذا الدليل، ستتعلم أساسيات SQL من الصفر مع التركيز على الجانب العملي والتكامل مع بايثون.

1. مقدمة إلى SQL

قواعد البيانات العلائقية (Relational Databases) تخزّن البيانات في جداول (Tables) تتكون من صفوف (Rows) و أعمدة (Columns). كل جدول يمثل كياناً معيناً (مثل المستخدمين أو المنتجات)، والعلاقات بين الجداول تُبنى باستخدام المفاتيح.

SQL هي اللغة التي تستخدمها للتواصل مع قاعدة البيانات. تنقسم أوامر SQL إلى عدة أقسام رئيسية:

  • DDL (Data Definition Language): إنشاء وتعديل هيكل الجداول — CREATE, ALTER, DROP
  • DML (Data Manipulation Language): التعامل مع البيانات — SELECT, INSERT, UPDATE, DELETE
  • DCL (Data Control Language): صلاحيات الوصول — GRANT, REVOKE
-- إنشاء جدول بسيط CREATE TABLE users ( id INTEGER PRIMARY KEY, name TEXT NOT NULL, email TEXT UNIQUE, age INTEGER ); -- إدراج بيانات INSERT INTO users (name, email, age) VALUES ('Ahmed', 'ahmed@example.com', 25);
🔍 شرح الكود:

الأمر CREATE TABLE ينشئ جدولاً جديداً بأعمدة محددة وأنواع بياناتها. PRIMARY KEY يحدد المفتاح الأساسي الذي يعرّف كل صف بشكل فريد. NOT NULL يمنع القيم الفارغة. INSERT INTO يُضيف صفاً جديداً إلى الجدول بالقيم المحددة.

2. SELECT - استعلام البيانات الأساسي

الأمر SELECT هو أكثر أوامر SQL استخداماً — وهو المسؤول عن جلب البيانات من قاعدة البيانات. يمكنك تحديد الأعمدة التي تريدها، وشروط التصفية، والترتيب، وغير ذلك.

-- اختيار كل الأعمدة SELECT * FROM users; -- اختيار أعمدة محددة SELECT name, email FROM users; -- الحد من النتائج SELECT * FROM users LIMIT 5; -- تجاهل القيم المكررة SELECT DISTINCT age FROM users;
🔍 شرح الكود:

SELECT * FROM يجلب كل الأعمدة من الجدول. لتحديد أعمدة معينة، اكتب أسماءها بدلاً من *. LIMIT يحدد عدد الصفوف المرتجعة. DISTINCT يُزيل القيم المكررة من النتائج.

3. WHERE - تصفية البيانات

جملة WHERE تُستخدم لتصفية الصفوف بناءً على شروط محددة. تدعم SQL مجموعة كبيرة من عوامل المقارنة والمنطق.

-- عوامل المقارنة الأساسية SELECT * FROM users WHERE age = 25; SELECT * FROM users WHERE age != 18; SELECT * FROM users WHERE age > 21; -- LIKE - مطابقة الأنماط SELECT * FROM users WHERE email LIKE '%@gmail.com'; -- IN - ضمن مجموعة SELECT * FROM users WHERE age IN (20, 25, 30); -- BETWEEN - ضمن نطاق SELECT * FROM users WHERE age BETWEEN 18 AND 35; -- IS NULL - القيم الفارغة SELECT * FROM users WHERE email IS NULL;
🔍 شرح الكود:

جملة WHERE تصفي النتائج حسب شرط معين. LIKE يسمح بمطابقة الأنماط باستخدام % (تطابق أي عدد من الأحرف) و _ (حرف واحد). IN تتحقق مما إذا كانت القيمة ضمن قائمة. BETWEEN تشمل القيم بين حدين. IS NULL تتحقق من القيم الفارغة.

4. ORDER BY - الترتيب

لترتيب النتائج تصاعدياً أو تنازلياً، استخدم ORDER BY متبوعاً باسم العمود.

-- ترتيب تصاعدي (افتراضي) SELECT name, age FROM users ORDER BY age ASC; -- ترتيب تنازلي SELECT name, age FROM users ORDER BY age DESC; -- ترتيب حسب أعمدة متعددة SELECT * FROM users ORDER BY age DESC, name ASC;
🔍 شرح الكود:

ORDER BY يرتب النتائج حسب العمود المحدد. ASC للترتيب التصاعدي (الأصغر إلى الأكبر — هذا هو الافتراضي). DESC للترتيب التنازلي (الأكبر إلى الأصغر). يمكنك الترتيب بعدة أعمدة بفصلها بفاصلة.

5. JOINs - ربط الجداول

أكثر ميزة قوية في SQL هي ربط الجداول ببعضها. JOIN يدمج صفوفاً من جدولين أو أكثر بناءً على علاقة بين أعمدة معينة.

لنفترض أن لدينا جدولين — users و orders:

-- INNER JOIN: فقط الصفوف المتطابقة في كلا الجدولين SELECT users.name, orders.product, orders.amount FROM users INNER JOIN orders ON users.id = orders.user_id; -- LEFT JOIN: كل الصفوف من الجدول الأيسر حتى لو لم يكن هناك تطابق SELECT users.name, orders.product FROM users LEFT JOIN orders ON users.id = orders.user_id; -- RIGHT JOIN: كل الصفوف من الجدول الأيمن SELECT users.name, orders.product FROM users RIGHT JOIN orders ON users.id = orders.user_id;
🔍 شرح الكود:

INNER JOIN يُرجع فقط الصفوف التي تطابق الشرط في كلا الجدولين. LEFT JOIN يُرجع كل صفوف الجدول الأيسر (users) والقيم المطابقة من الجدول الأيمن (orders) — والقيم غير المطابقة تكون NULL. RIGHT JOIN العكس: كل صفوف الجدول الأيمن.

-- مثال عملي: المستخدمون وطلباتهم مع الربط SELECT u.name, u.email, COUNT(o.id) AS total_orders, SUM(o.amount) AS total_spent FROM users u LEFT JOIN orders o ON u.id = o.user_id GROUP BY u.id ORDER BY total_spent DESC;
🔍 شرح الكود:

هذا الاستعلام يجمع بين LEFT JOIN و GROUP BY ودوال التجميع. نستخدم الأسماء المستعارة (u و o) لتبسيط الكتابة. COUNT(o.id) يحسب عدد الطلبات لكل مستخدم، و SUM(o.amount) يحسب المجموع. النتيجة تُرتب حسب أعلى قيمة إنفاق.

6. Aggregate Functions - دوال التجميع

دوال التجميع تُجري حسابات على مجموعة من الصفوف وتُعيد قيمة واحدة. غالباً ما تُستخدم مع GROUP BY لتقسيم النتائج إلى مجموعات.

-- COUNT: عدد الصفوف SELECT COUNT(*) AS total_users FROM users; -- SUM: مجموع القيم SELECT SUM(amount) AS revenue FROM orders; -- AVG: متوسط القيم SELECT AVG(age) AS avg_age FROM users; -- MIN / MAX: أدنى وأعلى قيمة SELECT MIN(age), MAX(age) FROM users; -- GROUP BY + HAVING SELECT user_id, COUNT(*) AS order_count FROM orders GROUP BY user_id HAVING order_count >= 3;
🔍 شرح الكود:

COUNT يحسب عدد الصفوف. SUM يجمع القيم الرقمية. AVG يحسب المتوسط. MIN و MAX ترجعان أدنى وأعلى قيمة. GROUP BY يُجمّع الصفوف حسب عمود معين. HAVING يُصفّي المجموعات (مثل WHERE لكن بعد التجميع — WHERE قبل التجميع، HAVING بعده).

7. Subqueries - الاستعلامات الفرعية

الاستعلام الفرعي (Subquery) هو استعلام داخل استعلام آخر. يمكنك استخدامه في جمل WHERE، أو FROM، أو حتى في SELECT نفسها.

-- Subquery في WHERE SELECT name FROM users WHERE id IN ( SELECT DISTINCT user_id FROM orders WHERE amount > 100 ); -- Subquery في FROM (استعلام فرعي كجدول مؤقت) SELECT avg_stats.avg_age FROM ( SELECT AVG(age) AS avg_age FROM users ) AS avg_stats WHERE avg_stats.avg_age > 30;
🔍 شرح الكود:

الاستعلام الفرعي المتصل (Correlated Subquery) هو استعلام فرعي يعتمد على قيم من الاستعلام الخارجي. يتم تنفيذه لكل صف في النتيجة الخارجية. في المثال الأول، نجد المستخدمين الذين لديهم طلبات بقيمة أكبر من 100. في المثال الثاني، نستخدم استعلاماً فرعياً في FROM كجدول مؤقت.

-- Correlated Subquery SELECT name, age FROM users u WHERE age > ( SELECT AVG(age) FROM users WHERE id != u.id );
🔍 شرح الكود:

هذا استعلام فرعي مرتبط (Correlated Subquery) — الاستعلام الداخلي يُنفذ لكل صف من الاستعلام الخارجي. كل صف من users u يقارن عمره بمتوسط أعمار باقي المستخدمين (id != u.id). هذا مثال متقدم لكنه قوي جداً.

8. CTE - Common Table Expressions

Common Table Expressions (CTE) تسمح لك بتعريف استعلامات مؤقتة بأسماء باستخدام جملة WITH. هذا يجعل الاستعلامات المعقدة أكثر قابلية للقراءة وإعادة الاستخدام.

-- CTE بسيط WITH active_users AS ( SELECT * FROM users WHERE age >= 18 ) SELECT name, age FROM active_users ORDER BY age DESC; -- CTEs متعددة WITH user_stats AS ( SELECT user_id, COUNT(*) AS orders, SUM(amount) AS total FROM orders GROUP BY user_id ), top_users AS ( SELECT * FROM user_stats WHERE total > 500 ) SELECT u.name, t.orders, t.total FROM top_users t JOIN users u ON t.user_id = u.id;
🔍 شرح الكود:

CTE تُعرِّف جدولاً مؤقتاً باسم يمكن الرجوع إليه في الاستعلام الرئيسي. باستخدام CTEs متعددة، يمكنك بناء استعلامات معقدة خطوة بخطوة. كل CTE مفصول بفاصلة، والأخير هو الاستعلام الرئيسي. هذا أفضل بكثير من الاستعلامات الفرعية المتداخلة.

-- Recursive CTE: تسلسل الأرقام WITH RECURSIVE numbers(n) AS ( SELECT 1 -- base case UNION ALL SELECT n + 1 FROM numbers WHERE n < 10 ) SELECT * FROM numbers; -- Recursive CTE: هيكل تنظيمي WITH RECURSIVE org_tree AS ( SELECT id, name, manager_id, 1 AS level FROM employees WHERE manager_id IS NULL UNION ALL SELECT e.id, e.name, e.manager_id, t.level + 1 FROM employees e JOIN org_tree t ON e.manager_id = t.id ) SELECT * FROM org_tree ORDER BY level, name;
🔍 شرح الكود:

Recursive CTE يستدعي نفسه لبناء بيانات هرمية أو متكررة. يتكون من جزأين: base case (نقطة البداية) و recursive step (الخطوة المتكررة) يفصل بينهما UNION ALL. المثال الأول يُنشئ تسلسل أرقام. الثاني يبني هيكلاً تنظيمياً من جدول الموظفين.

9. Indexes - الفهارس

الفهارس (Indexes) هي هياكل بيانات تسرّع استعلامات البحث في الجداول. تعمل مثل فهرس الكتاب — بدلاً من تصفح كل صفحة، تذهب مباشرة إلى الموقع المطلوب.

-- إنشاء فهرس CREATE INDEX idx_users_email ON users(email); -- فهرس مركب (على عدة أعمدة) CREATE INDEX idx_users_age_name ON users(age, name); -- فهرس فريد CREATE UNIQUE INDEX idx_users_username ON users(username); -- حذف فهرس DROP INDEX idx_users_email;
🔍 شرح الكود:

CREATE INDEX يُنشئ فهرساً على عمود معين لتسريع البحث والترتيب. الفهرس المركب على عدة أعمدة مفيد للاستعلامات التي تُصفّي بعدة أعمدة. الفهرس الفريد يضمن عدم وجود قيم مكررة في العمود. الفهارس تُسرّع القراءة لكنها تُبطئ الكتابة (INSERT/UPDATE).

-- تحليل أداء الاستعلام EXPLAIN ANALYZE SELECT * FROM users WHERE email = 'ahmed@example.com'; -- الاستعلام بدون فهرس: مسح كامل للجدل (Seq Scan) -- الاستعلام مع فهرس: بحث سريع عبر الفهرس (Index Scan)
🔍 شرح الكود:

EXPLAIN ANALYZE يُظهر خطة تنفيذ الاستعلام والوقت الفعلي المستغرق. بدون فهرس، تقوم قاعدة البيانات بمسح الجدول كاملاً (Seq Scan). مع الفهرس، تستخدم Index Scan وتجد الصف المطلوب بسرعة كبيرة — فرق قد يكون من ثوانٍ إلى أجزاء من الألف من الثانية.

10. Python + SQL - الاتصال من بايثون

يمكنك الاتصال بقواعد البيانات من بايثون باستخدام مكتبات مثل sqlite3 (مضمنة) أو psycopg2 (لـ PostgreSQL) أو mysql-connector. المبدأ واحد: إنشاء اتصال، تنفيذ استعلام، معالجة النتائج.

import sqlite3 # إنشاء اتصال بقاعدة بيانات (تُنشأ ملفاً إذا لم تكن موجودة) conn = sqlite3.connect('store.db') cursor = conn.cursor() # إنشاء جدول cursor.execute('''CREATE TABLE IF NOT EXISTS users ( id INTEGER PRIMARY KEY, name TEXT NOT NULL, email TEXT UNIQUE, age INTEGER )''') # إدراج بيانات مع عنصر نائب (parameterized query) cursor.execute( 'INSERT INTO users (name, email, age) VALUES (?, ?, ?)', ('Sara', 'sara@example.com', 28) ) conn.commit()
🔍 شرح الكود:

sqlite3.connect() يُنشئ اتصالاً بقاعدة البيانات. cursor() يُنشئ مؤشراً لتنفيذ الاستعلامات. الاستعلامات المParametrized تستخدم ? كعناصر نائبة للقيم — هذا يمنع هجمات SQL Injection تماماً. commit() يحفظ التغييرات.

# استعلام وجلب النتائج cursor.execute('SELECT * FROM users WHERE age > ?', (21,)) rows = cursor.fetchall() # list of tuples for row in rows: print(row) # fetchone - صف واحد فقط row = cursor.execute( 'SELECT * FROM users WHERE email = ?', ('sara@example.com',) ).fetchone() # إغلاق الاتصال conn.close()
🔍 شرح الكود:

fetchall() يُرجع كل الصفوف المطابقة كقائمة من tuples. fetchone() يُرجع الصف الأول فقط أو None إذا لم توجد نتائج. من المهم دائماً إغلاق الاتصال بـ close() لتحرير الموارد. يمكنك أيضاً استخدام fetchmany(n) لجلب عدد محدد من الصفوف.

# مثال مع PostgreSQL (psycopg2) # pip install psycopg2-binary import psycopg2 conn = psycopg2.connect( host='localhost', dbname='mydb', user='admin', password='secret' ) cursor = conn.cursor() # عناصر نائبة بـ %s (PostgreSQL) cursor.execute( 'INSERT INTO users (name, email) VALUES (%s, %s)', ('Khaled', 'khaled@example.com') ) conn.commit() cursor.close() conn.close()
🔍 شرح الكود:

مع psycopg2 (PostgreSQL) نستخدم %s كعناصر نائبة بدلاً من ?. الاتصال يتطلب host، dbname، user، password. المبدأ نفسه: execute() لتنفيذ الاستعلام، commit() لحفظ التغييرات. fetchall() و fetchone() تعملان بنفس الطريقة.

11. Transactions - المعاملات

المعاملات (Transactions) تسمح بتنفيذ مجموعة من العمليات مع ضمان أن كل العمليات تنجح أو تفشل معاً — لا حالة وسطية. هذا يضمن سلامة البيانات.

مبدأ ACID يضمن: Atomicity (الذرية — كل شيء أو لا شيء)، Consistency (الاتساق — البيانات تبقى صحيحة)، Isolation (العزل — المعاملات لا تتداخل)، Durability (الديمومة — التغييرات تبقى حتى بعد الأعطال).

-- SQL: معاملة صريحة BEGIN; UPDATE accounts SET balance = balance - 100 WHERE id = 1; UPDATE accounts SET balance = balance + 100 WHERE id = 2; COMMIT; -- أو ROLLBACK; للتراجع
🔍 شرح الكود:

BEGIN يبدأ معاملة جديدة. COMMIT يحفظ كل التغييرات في المعاملة. إذا حدث خطأ قبل COMMIT، يمكنك استخدام ROLLBACK للتراجع عن كل التغييرات. هذا يضمن أن تحويل الأموال مثلاً إما يتم بالكامل أو لا يتم أبداً — لا يمكن خصم المبلغ من طرف دون إضافته للطرف الآخر.

-- المعاملات في بايثون import sqlite3 conn = sqlite3.connect('bank.db') cursor = conn.cursor() try: # بدء المعاملة cursor.execute('BEGIN') # خصم من الحساب الأول cursor.execute( 'UPDATE accounts SET balance = balance - ? WHERE id = ?', (100, 1) ) # إضافة للحساب الثاني cursor.execute( 'UPDATE accounts SET balance = balance + ? WHERE id = ?', (100, 2) ) # تأكيد المعاملة — كل شيء أو لا شيء conn.commit() print('✅ التحويل تم بنجاح') except Exception as e: # تراجع عن كل التغييرات في حال حدوث خطأ conn.rollback() print(f'❌ فشل التحويل: {e}') finally: conn.close()
🔍 شرح الكود:

في بايثون، نبدأ معاملة بـ BEGIN ثم ننفذ العمليات داخل try. إذا نجحت كل العمليات، نستدعي commit() لحفظ التغييرات. إذا حدث أي خطأ، نستدعي rollback() في except للتراجع عن كل شيء. هذا يضمن أن قاعدة البيانات تبقى في حالة متسقة دائماً.

📝 ملخص SQL

في هذا الدليل تعلمت:

  • ما هي SQL وقواعد البيانات العلائقية والجداول والصفوف والأعمدة
  • جلب البيانات باستخدام SELECT, FROM, LIMIT, DISTINCT
  • تصفية البيانات بـ WHERE مع LIKE, IN, BETWEEN, IS NULL
  • ترتيب النتائج بـ ORDER BY ASC / DESC
  • ربط الجداول بـ INNER JOIN, LEFT JOIN, RIGHT JOIN
  • دوال التجميع COUNT, SUM, AVG, MIN, MAX مع GROUP BY و HAVING
  • الاستعلامات الفرعية (Subqueries) بأنواعها
  • Common Table Expressions (CTE) مع WITH — البسيط والمتكرر
  • الفهارس (Indexes) وتسريع الاستعلامات بـ EXPLAIN ANALYZE
  • الاتصال بقواعد البيانات من بايثون باستخدام sqlite3 و psycopg2
  • المعاملات (Transactions) ومبدأ ACID — BEGIN, COMMIT, ROLLBACK
💡 نصيحة ذهبية: تعلّم SQL استثمار رائع — هي لغة موجودة منذ 50 عاماً وستبقى لعقود. ابدأ بجداول بسيطة وانتقل تدريجياً للاستعلامات المعقدة. استخدم EXPLAIN ANALYZE لتحسين أداء استعلاماتك. والأهم: استخدم الاستعلامات المParametrized (? أو %s) لحماية تطبيقاتك من SQL Injection.
⚠️ خطأ شائع: نسيان WHERE مع UPDATE أو DELETE — هذا سيُعدّل أو يحذف كل الصفوف في الجدول! دائماً تأكد من وجود شرط WHERE قبل تنفيذ هذه الأوامر. أيضاً، لا تنسَ استدعاء commit() بعد التعديلات — وإلا ستفقد تغييراتك عند إغلاق الاتصال.
🏋️ تمرين: أنشئ قاعدة بيانات SQLite للمكتبة تحتوي على جدولين: books (id, title, author_id, year, price) و authors (id, name, country). اكتب استعلامات: (1) كل الكتب مع اسم المؤلف باستخدام JOIN. (2) المؤلفون الذين لديهم أكثر من كتاب مع COUNT. (3) أغلى كتاب لكل مؤلف مع MAX. (4) CTE يعرض الكتب التي صدرت بعد 2000 مع اسم المؤلف.
