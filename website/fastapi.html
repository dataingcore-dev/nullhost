FastAPI — بناء APIs سريعة وعصرية

إطار العمل الأسرع لبناء APIs بلغة بايثون مع توثيق تلقائي وأداء عالٍ

← العودة إلى الرئيسية

FastAPI هو إطار عمل حديث لبناء APIs بايثون. يتميز بالأداء العالي (ينافس Node.js و Go)، التوثيق التلقائي (Swagger UI)، ودعم async/await بشكل أصلي.

1. التثبيت والتشغيل

قبل أي شيء، ثبت المكتبات الضرورية. fastapi للإطار نفسه، و uvicorn لتشغيل السيرفر.

pip install fastapi uvicorn

لننشئ أول تطبيق. كل تطبيق FastAPI يبدأ بإنشاء كائن FastAPI() ثم تعريف endpoints.

# main.py — أول تطبيق FastAPI from fastapi import FastAPI app = FastAPI(title="My API", version="1.0.0") @app.get("/") def root(): return {"message": "Hello World"}
🔍 كيف يعمل؟ @app.get("/") يعني: عندما يأتي طلب GET للمسار /، نفذ الدالة root(). FastAPI يلقائياً يحوّل الـ dict إلى JSON.

لتشغيل التطبيق، استخدم الأمر التالي في الطرفية:

uvicorn main:app --reload

--reload يعيد تحميل السيرفر تلقائياً عند تغيير الكود (مفيد جداً أثناء التطوير). افتح المتصفح على http://127.0.0.1:8000 وسترى النتيجة. والأجمل — اذهب إلى /docs لترى واجهة Swagger التلقائية!

2. Path Parameters — معاملات المسار

يمكنك تمرير قيم متغيرة داخل المسار نفسه. مثلاً، نجلب عنصراً بواسطة ID. FastAPI يتحقق تلقائياً من نوع البيانات.

@app.get("/items/{item_id}") def read_item(item_id: int): return {"item_id": item_id}

جرب /items/42 → ستحصل على {"item_id": 42}. لو أدخلت نصاً مثل /items/abc، FastAPI سيرد بـ 422 Validation Error — لأنه يتوقع int.

3. Query Parameters — معاملات الاستعلام

المعاملات التي تأتي بعد ? في الرابط. مثلاً /users/?skip=0&limit=10 تُستخدم في pagination والبحث.

from typing import Optional @app.get("/users/") def list_users( skip: int = 0, # قيمة افتراضية 0 limit: int = 10, # قيمة افتراضية 10 search: Optional[str] = None # اختياري ): return {"skip": skip, "limit": limit, "search": search}
💡 الفرق بين Path و Query: إذا كان المتغير جزءاً من المسار نفسه (مثل /items/5) فهو Path Parameter. إذا جاء بعد ? (مثل /items/?skip=5) فهو Query Parameter.

4. Request Body — إرسال بيانات (POST)

لإرسال بيانات من العميل إلى السيرفر، استخدم POST مع نموذج Pydantic. Pydantic يتحقق من صحة البيانات تلقائياً.

from pydantic import BaseModel class Item(BaseModel): name: str price: float is_offer: bool = False # قيمة افتراضية @app.post("/items/") def create_item(item: Item): return {"name": item.name, "price_with_tax": item.price * 1.15}

أرسل JSON كالتالي في الـ body:

{ "name": "Laptop", "price": 1000.0 }

FastAPI سيتحقق أن name موجود (إلزامي) و price رقم صحيح. لو أرسلت بيانات ناقصة، ستحصل على خطأ 422 مع رسالة توضح المشكلة.

5. CRUD كامل — إنشاء، قراءة، تحديث، حذف

هذا مثال متكامل لـ API يدير عناصر مع جميع العمليات الأساسية.

from typing import List from fastapi import HTTPException items_db: List[Item] = [] # قاعدة بيانات مؤقتة (في الذاكرة) @app.post("/items/", response_model=Item) # إنشاء def create(item: Item): items_db.append(item) return item @app.get("/items/", response_model=List[Item]) # قراءة الكل def list_all(): return items_db @app.get("/items/{item_id}", response_model=Item) # قراءة واحد def get_one(item_id: int): if item_id >= len(items_db): raise HTTPException(status_code=404, detail="Item not found") return items_db[item_id] @app.delete("/items/{item_id}") # حذف def delete(item_id: int): if item_id >= len(items_db): raise HTTPException(status_code=404) items_db.pop(item_id) return {"message": "Deleted"}
⚠️ خطأ شائع: نسيان response_model. بدونها، FastAPI قد يرجع حقولاً إضافية أو يعرض بيانات حساسة. حدد response_model دائماً.

6. Dependency Injection — حقن التبعيات

بدلاً من تكرار نفس المعاملات في كل endpoint، استخدم Depends لاستخراجها. هذا يجعل الكود أنظف وأسهل للاختبار.

from fastapi import Depends def common_params(q: Optional[str] = None, skip: int = 0, limit: int = 10): return {"q": q, "skip": skip, "limit": limit} @app.get("/search/") def search(params: dict = Depends(common_params)): return params

أي endpoint يستخدم Depends(common_params) سيحصل تلقائياً على q و skip و limit من الـ query string. مثلاً /search/?q=laptop&skip=0&limit=5.

🔧 استخدامات أخرى لـ Depends: التحقق من المصادقة، الاتصال بقاعدة البيانات، الحصول على المستخدم الحالي.

7. Async — الأداء العالي

FastAPI يدعم async/await بشكل أصلي. استخدمه للعمليات المنتظرة (I/O): طلبات HTTP خارجية، استعلامات قاعدة بيانات، قراءة ملفات.

import asyncio @app.get("/slow/") async def slow_endpoint(): await asyncio.sleep(2) # يحاكي انتظار API خارجي return {"message": "Completed"}

متى تستخدم async def ومتى def؟

  • async def → عمليات I/O: طلبات HTTP، استعلامات DB، قراءة ملفات
  • def عادي → عمليات CPU: حسابات، معالجة بيانات، تحويلات

FastAPI يتعامل مع الحالتين بكفاءة. لو استخدمت def لعمليات I/O، السيرفر سينتظر بدون فائدة.

8. CORS — مشاركة الموارد

إذا كان تطبيق الواجهة (Frontend) يعمل على دومين مختلف عن API، ستحتاج إلى تفعيل CORS. مثلاً التطبيق على localhost:3000 و API على localhost:8000.

from fastapi.middleware.cors import CORSMiddleware app.add_middleware( CORSMiddleware, allow_origins=["*"], # للإنتاج: ["https://mysite.com"] allow_credentials=True, allow_methods=["*"], allow_headers=["*"], )
⚠️ مهم: في الإنتاج، لا تستخدم allow_origins=["*"]. حدد الدومينات المسموحة فقط. وضّح ذلك في ملف config.py.

9. الاتصال بقاعدة بيانات PostgreSQL

لتوصيل FastAPI بقاعدة بيانات، استخدم psycopg2 مع Depends. بهذه الطريقة، الاتصال يُفتح عند كل طلب ويُغلق تلقائياً بعد الانتهاء.

import psycopg2 from psycopg2.extras import RealDictCursor DATABASE_URL = "postgresql://user:pass@localhost:5432/mydb" def get_db(): conn = psycopg2.connect(DATABASE_URL, cursor_factory=RealDictCursor) try: yield conn finally: conn.close() @app.get("/users/") def get_users(db=Depends(get_db)): cursor = db.cursor() cursor.execute("SELECT id, name, email FROM users") return cursor.fetchall()

ماذا يحدث هنا خطوة بخطوة:

  1. عند طلب GET /users/، FastAPI يستدعي get_db()
  2. يُفتح اتصال PostgreSQL (باستخدام RealDictCursor لتحويل النتائج لـ dict)
  3. يُنفذ الاستعلام ويُعيد النتائج
  4. عند انتهاء الطلب، finally يغلق الاتصال تلقائياً

10. أفضل الممارسات

  • استخدم Routers: قسّم endpoints إلى ملفات منفصلة باستخدام APIRouter
  • Pydantic للخروج أيضاً: استخدم response_model لضمان شكل الـ response
  • Environment Variables: خزّن DATABASE_URL و SECRET_KEY في متغيرات بيئة
  • اختبار: استخدم TestClient لاختبار endpoints
  • توثيق: أضف وصفاً لكل endpoint (description, summary, tags)
