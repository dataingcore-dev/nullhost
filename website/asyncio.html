AsyncIO - البرمجة غير المتزامنة

Asynchronous Programming in Python — Coroutines, Event Loop, and Concurrency

مقدمة إلى AsyncIO

AsyncIO هي مكتبة قياسية في بايثون (موجودة من Python 3.4) تتيح لك كتابة كود غير متزامن (asynchronous) باستخدام نموذج async/await. البرمجة غير المتزامنة تسمح لبرنامجك بالقيام بمهام متعددة في نفس الوقت دون الحاجة إلى خيوط تنفيذ متعددة (threads). الفكرة الأساسية: بدلاً من انتظار اكتمال مهمة بطيئة (مثل قراءة ملف أو طلب HTTP)، يمكن للبرنامج تنفيذ مهام أخرى أثناء الانتظار.

متى نستخدم AsyncIO؟ عندما يكون البرنامج I/O bound — أي محدود بعمليات الإدخال/الإخراج مثل طلبات الشبكة أو قراءة الملفات أو قواعد البيانات. لا يُنصح باستخدامه للمهام CPU-bound (المهام التي تستهلك المعالج مثل العمليات الحسابية الثقيلة)، فهذه تحتاج إلى multiprocessing أو threading.

💡 الفرق بين AsyncIO و Threading: في الـ threading، نظام التشغيل هو من يُقرر متى يبدّل بين الخيوط (preemptive). في AsyncIO، البرنامج نفسه يُقرر متى يبدّل بين المهام (cooperative) باستخدام await. هذا يعني أن AsyncIO أكثر كفاءة في استهلاك الذاكرة ويمكنه إدارة آلاف المهام المتزامنة بسهولة — بينما الـ threads محدودة بعدد أنوية المعالجة وتستهلك ذاكرة أكبر لكل خيط.
Coroutines - الكوروتينات

الكوروتين (coroutine) هو دالة غير متزامنة تُعرّف باستخدام async def. عند استدعاء كوروتين، لا يتم تنفيذه فوراً — بل يُعيد كائن كوروتين (coroutine object) يحتاج إلى await أو تشغيله عبر asyncio.run(). الكلمة المفتاحية await تسلّم التحكم إلى event loop لتشغيل مهام أخرى أثناء انتظار اكتمال العملية البطيئة.

import asyncio # تعريف كوروتين باستخدام async def async def say_hello(): print("مرحباً!") await asyncio.sleep(1) # انتظر ثانية دون حجب البرنامج print("انتهى الانتظار!") # تشغيل الكوروتين asyncio.run(say_hello())
شرح الكود:

نستخدم async def لتعريف دالة غير متزامنة تُسمى كوروتين. داخل الكوروتين، نستخدم await قبل العمليات البطيئة — مثل asyncio.sleep(1) — لتسليم التحكم إلى الـ event loop. الدالة asyncio.run() هي الطريقة المُوصى بها لتشغيل أي كود غير متزامن؛ تنشئ event loop جديد، تشغّل الكوروتين، ثم تُغلقه.

لاحظ أن استخدام time.sleep() العادية داخل كوروتين سيحجب الـ event loop بالكامل — وهذا خطأ شائع! استخدم دائماً asyncio.sleep() بدلاً منها.

Event Loop

Event Loop (حلقة الأحداث) هو قلب AsyncIO. هو عبارة عن حلقة لا نهائية تدير تنفيذ الكوروتينات والمهام. عندما تستخدم await، يعود التحكم إلى الـ event loop الذي يبحث عن مهام أخرى جاهزة للتشغيل. عندما تنجز العملية البطيئة، يُستأنف الكوروتين من حيث توقف.

في الإصدارات القديمة من بايثون، كنا نستخدم asyncio.get_event_loop() ثم loop.run_until_complete(). لكن من Python 3.7 فصاعداً، asyncio.run() هي الطريقة المُوصى بها لأنها تتعامل مع إنشاء وإغلاق الـ loop تلقائياً.

# الطريقة القديمة (Python 3.6) loop = asyncio.get_event_loop() loop.run_until_complete(say_hello()) loop.close() # الطريقة الحديثة (Python 3.7+) asyncio.run(say_hello())
شرح الكود:

الـ Event Loop هو المسؤول عن جدولة وتشغيل الكوروتينات. asyncio.run() هي واجهة عالية المستوى: تنشئ loop، تشغّل الكوروتين، وتُغلق الـ loop بعد الانتهاء. في الإصدارات القديمة، كنا نضطر لإدارة الـ loop يدوياً — وهو مصدر شائع للأخطاء (مثل نسيان إغلاق الـ loop).

💡 ملاحظة مهمة: asyncio.run() لا يمكن استدعاؤها من داخل كوروتين قيد التشغيل. إذا كنت داخل كوروتين وتحتاج لتشغيل كوروتين آخر، استخدم await مباشرة. يوجد loop واحد فقط لكل thread.
Tasks - المهام

الـ Task هو كائن يلفّ كوروتين ويُجدول تنفيذه في الـ event loop. باستخدام asyncio.create_task()، يمكنك تشغيل كوروتين في الخلفية (في الخلفية هنا يعني بالتوازي مع مهام أخرى). هذا يسمح بتشغيل عدة عمليات غير متزامنة بشكل متزامن (concurrently).

import asyncio async def task_a(): print("بدء المهمة A") await asyncio.sleep(2) print("اكتملت المهمة A") return "نتيجة A" async def task_b(): print("بدء المهمة B") await asyncio.sleep(1) print("اكتملت المهمة B") return "نتيجة B" async def main(): # إنشاء مهام متزامنة t1 = asyncio.create_task(task_a()) t2 = asyncio.create_task(task_b()) # انتظار اكتمال كل المهام r1 = await t1 r2 = await t2 print(f"النتائج: {r1}, {r2}") asyncio.run(main())
شرح الكود:

asyncio.create_task() تُنشئ Task جديداً يُجدول الكوروتين للتشغيل في الخلفية. هذا يسمح بتشغيل مهمة (A) ومهمة (B) بشكل متزامن — لا تنتظر إحداهما الأخرى. ستلاحظ أن B تبدأ بعد A مباشرة وتنتهي أولاً لأنها تنام لمدة أقل. الإجمالي الزمني سيكون ~2 ثانية وليس 3 (ولو كنا استخدمنا await مباشرة لكانت 3 ثوانٍ).

asyncio.gather

الدالة asyncio.gather() تسمح بتشغيل عدة كوروتينات بالتوازي (concurrently) وانتظار اكتمالها جميعاً. تُعيد قائمة بالنتائج بنفس ترتيب الكوروتينات المُمرّرة. الخيار return_exceptions=True يجعلها تتعامل مع الاستثناءات كقيم مُرجعة بدلاً من رفعها فوراً.

import asyncio async def fetch_data(url, delay): print(f"جارٍ جلب: {url}") await asyncio.sleep(delay) print(f"تم جلب: {url}") return f"بيانات {url}" async def main(): results = await asyncio.gather( fetch_data("site1.com", 3), fetch_data("site2.com", 2), fetch_data("site3.com", 1), return_exceptions=True ) print(f"جميع النتائج: {results}") asyncio.run(main())
شرح الكود:

asyncio.gather() تشغّل كل الكوروتينات المُمرّرة بشكل متزامن وتنتظر اكتمالها جميعاً. المدة الإجمالية = أطول مدة بينها (3 ثوانٍ)، وليس مجموعها (6 ثوانٍ). return_exceptions=True يعني: إذا فشل أحد الكوروتينات، لا تُرفع الـ exception فوراً بل تُعاد كقيمة في القائمة — يمكنك معالجتها لاحقاً.

Async Context Managers

مدير السياق غير المتزامن (async context manager) يستخدم async with بدلاً من with العادي. يُستخدم لإدارة الموارد التي تحتاج عمليات إعداد وإغلاق غير متزامنة — مثل فتح وإغلاق اتصال بقاعدة بيانات أو ملف. يمكنك تنفيذ واحد بنفسك باستخدام __aenter__ و __aexit__.

import asyncio class AsyncResource: async def __aenter__(self): print("فتح المورد...") await asyncio.sleep(1) return "المورد جاهز" async def __aexit__(self, exc_type, exc_val, exc_tb): print("إغلاق المورد...") await asyncio.sleep(0.5) async def main(): async with AsyncResource() as resource: print(f"استخدام: {resource}") asyncio.run(main())
شرح الكود:

مدير السياق غير المتزامن يُنفّذ __aenter__ (يُستدعى عند الدخول إلى async with) و __aexit__ (يُستدعى عند الخروج). كلاهما كوروتينات — يمكن استخدام await داخلهما. هذا مثالي لإدارة اتصالات قواعد البيانات، جلسات HTTP، والملفات التي تحتاج فتحاً وإغلاقاً غير متزامنين.

مكتبات مثل aiohttp و asyncpg توفر مديري سياق غير متزامنين جاهزين لإدارة جلسات العمل والاتصالات.

Async Iterators

الـ مُكرّر غير المتزامن (async iterator) يسمح بالتكرار على عناصر تُنتج بشكل غير متزامن. تستخدم async for للتكرار عليه. لتنفيذ واحد بنفسك، تحتاج إلى تعريف __aiter__ (يُعيد self) و __anext__ (يُعيد العنصر التالي أو يرفع StopAsyncIteration).

الطريقة الأسهل هي استخدام async generator — دالة مولّد غير متزامنة تستخدم async for مع yield:

import asyncio # Async Generator — أسهل طريقة async def countdown(start): while start > 0: await asyncio.sleep(1) yield start start -= 1 async def main(): async for num in countdown(5): print(f"العد التنازلي: {num}") asyncio.run(main())
شرح الكود:

المُولّد غير المتزامن (async generator) يُعرّف بـ async def ويحتوي على yield. كل تكرار باستخدام async for ينتظر await داخل المُولّد قبل إنتاج القيمة التالية. هذا مثالي لقراءة البيانات من مصدر بطيء (مثل API أو قاعدة بيانات) صفاً تلو الآخر.

asyncio.wait_for والـ Timeouts

عند التعامل مع عمليات خارجية (مثل طلبات HTTP أو اتصالات قاعدة بيانات)، من الضروري تحديد مهلة زمنية (timeout) لمنع تعليق البرنامج إلى الأبد. الدالة asyncio.wait_for() تغلّف كوروتين بمهلة زمنية — إذا لم يكتمل خلال المدة المُحددة، ترفع asyncio.TimeoutError.

import asyncio async def slow_operation(): print("بدء العملية البطيئة...") await asyncio.sleep(5) return "نتيجة" async def main(): try: result = await asyncio.wait_for( slow_operation(), timeout=2 ) print(f"النتيجة: {result}") except asyncio.TimeoutError: print("❌ تجاوزت المهلة الزمنية!") asyncio.run(main())
شرح الكود:

asyncio.wait_for() تلفّ كوروتيناً بمُؤقّت. إذا لم يكتمل الكوروتين خلال timeout ثانية، تُرفع asyncio.TimeoutError. هذا ضروري في التطبيقات الحقيقية — تخيل أن خادم API لا يستجيب أبداً. الـ timeout يضمن أن برنامجك لن يُعلق للأبد، ويمكنك التعامل مع الحالة برشاقة.

💡 نصيحة: استخدم دائماً asyncio.wait_for() مع أي عملية شبكة أو قاعدة بيانات. اختر timeout معقول — طويل كفاية للعملية الشرعية، لكن قصير كفاية لعدم تعليق المستخدم. 5-30 ثانية هي مدة شائعة حسب الحالة.
asyncio.to_thread

أحياناً تحتاج لتشغيل كود متزامن (synchronous) — مثل دالة time.sleep() أو مكتبة غير متوافقة مع AsyncIO — دون حجب الـ event loop. الدالة asyncio.to_thread() (مضافة في Python 3.9) تشغّل الكود المتزامن في thread pool منفصل، مما يمنع حجب الـ event loop.

import asyncio import time def blocking_io(): # محاكاة عملية I/O متزامنة (مثل قراءة ملف أو طلب HTTP بمكتبة غير async) print("بدء عملية متزامنة...") time.sleep(3) # هذا سيحجب الخيط العادي! return "بيانات من عملية متزامنة" async def main(): print("قبل to_thread") result = await asyncio.to_thread(blocking_io) print(f"بعد to_thread: {result}") asyncio.run(main())
شرح الكود:

asyncio.to_thread() تشغّل دالة متزامنة في thread pool منفصل. هذا يسمح للـ event loop بمواصلة تشغيل مهام أخرى أثناء انتظار اكتمال الدالة المتزامنة. استخدمها عندما تضطر لاستخدام مكتبة غير غير متزامنة (مثل requests بدلاً من aiohttp) أو للتعامل مع عمليات I/O لا تدعم async.

asyncio.to_thread() تستخدم loop.run_in_executor(None, func, ...) داخلياً. الفرق أن to_thread() أبسط وتُستخدم للـ default thread pool.

مثال عملي: طلبات HTTP غير متزامنة

من أكثر الاستخدامات شيوعاً لـ AsyncIO هو إرسال طلبات HTTP متعددة بشكل متزامن لتسريع جلب البيانات من APIs. بدلاً من إرسال طلب والانتظار، ثم إرسال الثاني، يمكنك إرسالها كلها معاً والانتظار لجميع النتائج. المكتبة الأشهر لذلك هي aiohttp.

import asyncio import aiohttp async def fetch_url(session, url): async with session.get(url) as response: return await response.text() async def main(): urls = [ "https://jsonplaceholder.typicode.com/posts/1", "https://jsonplaceholder.typicode.com/posts/2", "https://jsonplaceholder.typicode.com/posts/3", ] async with aiohttp.ClientSession() as session: tasks = [asyncio.create_task(fetch_url(session, url)) for url in urls] results = await asyncio.gather(*tasks) for url, html in zip(urls, results): print(f"{url}: {len(html)} حرف") asyncio.run(main())
شرح الكود:

هذا النمط الأساسي للعمل مع طلبات HTTP غير متزامنة: ننشئ جلسة (session) باستخدام async with، نُنشئ قائمة مهام (tasks) لكل URL، نجمع النتائج بـ asyncio.gather(). بدلاً من انتظار كل طلب 0.5 ثانية بالتسلسل (إجمالي 1.5 ثانية لـ 3 طلبات)، تعمل الطلبات بشكل متزامن — كلها تنتهي في ~0.5 ثانية.

💡 ملاحظة: إذا لم تكن تستطيع تثبيت aiohttp، يمكنك استخدام httpx مع httpx.AsyncClient() — واجهة مشابهة جداً. أو استخدم asyncio.to_thread() مع مكتبة requests العادية للحل السريع.
أخطاء شائعة (Common Pitfalls)

AsyncIO رائع لكنه يأتي مع مجموعة من الأخطاء التي يقع فيها المبتدئون وحتى المطورون ذوو الخبرة. إليك أبرزها:

1. عدم استخدام await مع كوروتين

هذا أكثر خطأ شائع. عندما تستدعي كوروتين بدون await، لا يتم تشغيله — بل تحصل على كائن كوروتين (<coroutine object>) ولن يشتغل أبداً.

# ❌ خطأ: نسيان await async def foo(): return 42 foo() # هذا لا يفعل شيئاً! يُعيد كائن كوروتين # ✅ صحيح: result = await foo() print(result)
شرح الكود:

عند استدعاء دالة مُعرّفة بـ async def، تُعيد كائن كوروتين — لا يتم تنفيذ أي كود داخلها حتى تستخدم await. نسيان await لا يُسبب خطأ (إلا إذا تجاهلت التحذير RuntimeWarning: coroutine never awaited) — الكوروتين يبقى معلقاً إلى الأبد!

2. حجب الـ Event Loop

استخدام دوال متزامنة تحجب الخيط (مثل time.sleep() أو حلقات حسابية طويلة) تجمّد الـ event loop بأكمله — لا يمكن تشغيل أي مهمة أخرى حتى تنتهي العملية.

# ❌ خطأ: حجب الـ event loop async def bad(): import time time.sleep(5) # يجمّد كل شيء! print("انتهى") # ✅ صحيح: استخدام asyncio.sleep async def good(): await asyncio.sleep(5) print("انتهى") # ✅ للكود المتزامن الاضطراري: استخدم to_thread async def better(): result = await asyncio.to_thread(time.sleep, 5) print("انتهى")
شرح الكود:

الـ Event Loop يعمل في خيط واحد. أي كود متزامن يحجب هذا الخيط يوقف جميع المهام غير المتزامنة الأخرى. استخدم دائماً بدائل AsyncIO مثل asyncio.sleep(). إذا اضطررت لاستخدام كود متزامن، لفه بـ asyncio.to_thread() ليُشغّل في خيط منفصل.

3. خلط الكود المتزامن وغير المتزامن

لا يمكن استدعاء كوروتين من داخل دالة عادية (غير async) بدون استخدام asyncio.run(). ولا يمكن استخدام await خارج async def.

# ❌ خطأ: await خارج async def def regular_func(): result = await some_coroutine() # SyntaxError! # ✅ صحيح: استخدم asyncio.run في الدوال العادية def regular_func(): result = asyncio.run(some_coroutine()) # ❌ خطأ: استدعاء asyncio.run داخل كوروتين قيد التشغيل async def nested(): await asyncio.run(another_coro()) # RuntimeError! # ✅ صحيح: await مباشرة async def nested(): await another_coro()
شرح الكود:

هناك قاعدة ذهبية: await يعمل فقط داخل async def. asyncio.run() يعمل فقط خارج أي كوروتين قيد التشغيل — يُستخدم مرة واحدة كنقطة دخول للبرنامج غير المتزامن. إذا كنت داخل كوروتين وتحتاج لتشغيل كوروتين آخر، استخدم await مباشرة.

4. استخدام async مع دوال غير مناسبة

AsyncIO لا يُسرّع الـ CPU-bound code. إذا كان لديك عملية حسابية ثقيلة (مثل معالجة صور أو تحليل بيانات ضخمة)، فإن تشغيلها في كوروتين لن يفيد — بل قد يبطئ الأمور بسبب الـ overhead. للمهام الحسابية، استخدم multiprocessing أو concurrent.futures.ProcessPoolExecutor.

💡 القاعدة البسيطة: إذا كانت مهمتك تنتظر شيئاً (شبكة، ملف، قاعدة بيانات) → استخدم AsyncIO. إذا كانت مهمتك تحسب شيئاً (حلقات، عمليات رياضية، معالجة بيانات) → استخدم الـ threads أو الـ processes.

5. عدم التعامل مع الاستثناءات في المهام

إذا رفع كوروتين قيد التشغيل (كـ Task) استثناءً ولم يتم التعامل معه، سيُخزّن الاستثناء في الـ Task. عند استدعاء await على الـ Task، سيُعاد رفع الاستثناء. إذا لم تستخدم await أبداً، سيظهر الاستثناء كـ RuntimeWarning عند جمع القمامة (garbage collection).

# ❌ خطر: استثناء غير مُعالَج في مهمة خلفية async def main(): task = asyncio.create_task(faulty_coro()) # لم نستخدم await task — قد نفتقد الاستثناء! # ✅ صحيح: التعامل مع الاستثناءات async def main(): task = asyncio.create_task(faulty_coro()) try: result = await task except Exception as e: print(f"حدث خطأ: {e}")
شرح الكود:

عند إنشاء Task بـ create_task()، أي استثناء داخله يُخزّن في كائن الـ Task. إذا لم تستخدم await task (أو task.exception())، قد لا تكتشف الفشل أبداً. استخدم await على مهامك الخلفية في نقطة ما، أو استخدم task.add_done_callback() للتعامل معها.

📝 ملخص AsyncIO

في هذا الدليل تعلمت:

  • البرمجة غير المتزامنة — تُستخدم للمهام I/O bound (الشبكة، الملفات، قواعد البيانات)
  • الكوروتينات (Coroutines) — تُعرّف بـ async def وتُشغّل بـ await
  • Event Loop — القلب النابض لـ AsyncIO، يدير جدولة المهام
  • المهام (Tasks) — asyncio.create_task() لتشغيل كوروتينات بالخلفية
  • asyncio.gather — تشغيل عدة كوروتينات بالتوازي وجمع النتائج
  • مديري السياق غير المتزامنين — async with مع __aenter__/__aexit__
  • المُكرّرات غير المتزامنة — async for مع async generators
  • التايم آوت — asyncio.wait_for() و asyncio.TimeoutError
  • to_thread — تشغيل كود متزامن دون حجب الـ event loop
  • الأخطاء الشائعة — نسيان await، حجب الـ loop، خلط sync/async
💡 نصيحة أخيرة: ابدأ ببطء. اكتب كوروتيناً واحداً بسيطاً باستخدام asyncio.sleep(). ثم جرب create_task() مع مهمتين. ثم انتقل إلى gather(). لا تحاول فهم كل شيء دفعة واحدة — AsyncIO يحتاج بعض الوقت لتستوعب طريقة التفكير غير المتزامنة. الخطأ الأكثر فائدة يمكنك فعله: انس await مرة واحدة — ستفهم الدرس للأبد! 😄
🏋️ تمرين عملي: اكتب برنامجاً غير متزامناً يقوم بالآتي:
1. يُعرِّف 3 كوروتينات، كل واحد يطبع اسمه وينتظر مدة مختلفة (1، 2، 3 ثوانٍ)
2. يشغّلها جميعاً باستخدام asyncio.gather()
3. يقيس الوقت الإجمالي باستخدام time.perf_counter()
4. أضف كوروتين رابع يرفع استثناءً واستخدم return_exceptions=True
🔍 هل الوقت الإجمالي ~3 ثوانٍ أم ~6 ثوانٍ؟ لماذا؟
