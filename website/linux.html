Linux - سطر الأوامر للمبرمج

دليل شامل لأوامر Linux الأساسية لمطوري بايثون — من navigation إلى networking

← العودة إلى الرئيسية

سطر أوامر Linux (الـ Terminal) هو واجهة قوية تمنحك تحكماً كاملاً بالنظام. كمبرمج بايثون، ستستخدمه يومياً — تشغيل السكريبتات، إدارة الملفات، مراقبة العمليات، والاتصال بالخوادم. هذا الدليل يغطي كل ما تحتاجه لتصبح proficient في CLI.

1. Terminal - الواجهة الطرفية

الـ Terminal هو برنامج يفسر الأوامر التي تكتبها وينفذها. عندما تفتحه، ترى Shell Prompt — عادةً شيئاً مثل user@host:~$. هذا يعني أن الـ Shell جاهز لاستقبال أمرك.

التركيب العام للأمر في Linux:

command [options] [arguments]
🔍 شرح الكود:

كل أمر في Linux يتكون من ثلاثة أجزاء: الأمر (command) نفسه مثل ls أو cd، متبوعاً بـ خيارات (options) تبدأ عادةً بشرطة - لتعديل سلوك الأمر، ثم الوسائط (arguments) مثل أسماء الملفات أو المسارات. الأقواس المربعة [] تعني أن الجزء اختياري.

افتح الـ Terminal من قائمة التطبيقات أو باستخدام الاختصار Ctrl+Alt+T على Ubuntu/Debian. غالباً ما ترى شيئاً مثل:

ahmed@ahmed-pc:~$ # هذا هو الـ prompt
🔍 شرح الكود:

الـ prompt يظهر اسم المستخدم (ahmed) بعدها @ ثم اسم الجهاز (ahmed-pc)، ثم : والدليل الحالي (~ يعني home)، وأخيراً $ للمستخدم العادي أو # للمستخدم الجذر (root). كل ما تكتبه يظهر بعد الـ $ وينفذ عند الضغط على Enter.

2. Navigation - التنقل

التنقل في نظام الملفات هو أول مهارة يجب أن تتقنها. إليك الأوامر الأساسية:

pwd - أين أنا؟

pwd # Output: /home/ahmed/projects
🔍 شرح الكود:

الأمر pwd (اختصار Print Working Directory) يطبع المسار الكامل (absolute path) للدليل الذي أنت فيه حالياً. هذا مفيد جداً عندما تتنقل بين مجلدات متعددة وتريد التأكد من موقعك.

ls - سرد الملفات

ls # سرد الملفات والمجلدات في الدليل الحالي ls -l # عرض بتفصيل (permissions, size, date) ls -a # عرض الملفات المخفية (التي تبدأ بـ .) ls -la # دمج الخيارين — تفصيل مع ملفات مخفية ls -lh # أحجام الملفات بشكل readable (K, M, G)
🔍 شرح الكود:

الأمر ls (اختصار List) يسرد محتويات الدليل. الخيار -l يعرض تفاصيل مثل الصلاحيات وعدد الروابط والمالك والحجم والتاريخ. -a (all) يظهر الملفات المخفية (تبدأ بنقطة). يمكن دمج الخيارات: -la = -l -a. -h يجعل الأحجام مقروءة (K, M, G).

cd - تغيير الدليل

cd /home/ahmed/projects # الانتقال إلى مسار مطلق cd Documents # الانتقال إلى مجلد فرعي (مسار نسبي) cd .. # الرجوع إلى المجلد الأب cd ../.. # الرجوع مجلدين للأعلى cd ~ # الذهاب إلى home directory cd - # الرجوع إلى آخر دليل كنت فيه cd . # يبقيك في نفس الدليل (عديم الفائدة)
🔍 شرح الكود:

cd (Change Directory) هو أمر التنقل الأساسي. المسار المطلق (absolute path) يبدأ من الجذر /. المسار النسبي (relative path) يعتمد على موقعك الحالي. النقطة . تعني الدليل الحالي، و .. تعني الدليل الأب. ~ اختصار لمجلد المستخدم الرئيسي (home). cd - يعود إلى آخر دليل.

المسارات المطلقة والنسبية

فهم الفرق بين المسار المطلق والنسبي أساسي:

pwd # /home/ahmed/projects/myapp # مسار مطلق — يبدأ من / cd /home/ahmed/Documents # مسار نسبي — يعتمد على موقعك الحالي cd ../../../etc # يصعد 3 مجلدات ثم يدخل etc
🔍 شرح الكود:

المسار المطلق يبدأ من جذر النظام / ويعمل من أي مكان. المسار النسبي يعتمد على الدليل الحالي — يستخدم .. للصعود للأعلى. مثلاً من /home/ahmed/projects/myapp، المسار ../../../etc يصعد: myapp → projects → ahmed → home، ثم يدخل etc، أي /etc.

3. File Operations - عمليات الملفات

إدارة الملفات والمجلدات هي صميم العمل على Linux. هذه الأوامر ستستخدمها يومياً:

touch - إنشاء ملف

touch main.py # إنشاء ملف فارغ touch main.py utils.py config.py # إنشاء عدة ملفات ls -l main.py # التحقق من تاريخ التعديل
🔍 شرح الكود:

touch ينشئ ملفاً فارغاً إذا لم يكن موجوداً، أو يُحدّث تاريخ التعديل (timestamp) إذا كان موجوداً. مفيد جداً لإنشاء ملفات سريعة — مثلاً ملفات Python جديدة لمشروعك. يمكن إنشاء عدة ملفات في أمر واحد بفصلها بمسافة.

mkdir - إنشاء مجلدات

mkdir my_project # إنشاء مجلد واحد mkdir -p my_project/src/utils # إنشاء هيكل مجلدات متداخلة mkdir -p {tests,docs,scripts} # إنشاء عدة مجلدات مرة واحدة
🔍 شرح الكود:

mkdir (Make Directory) ينشئ مجلدات. الخيار -p (parents) ينشئ المجلدات الوسيطة تلقائياً — إذا كان src غير موجود، سيُنشئه ثم ينشئ utils داخله. هذا يمنع أخطاء "No such file or directory".

cp - نسخ

cp file.txt backup.txt # نسخ ملف cp -r my_project/ my_project_backup/ # نسخ مجلد (r = recursive) cp -i file.txt dest/ # تفعيل التأكيد قبل الكتابة
🔍 شرح الكود:

cp (Copy) ينسخ الملفات والمجلدات. لنسخ مجلد، استخدم -r (recursive) لنسخ كل محتوياته. -i (interactive) يطلب تأكيداً قبل الكتابة فوق ملف موجود. الصيغة العامة: cp [options] source destination.

mv - نقل/إعادة تسمية

mv file.txt /tmp/ # نقل ملف إلى مجلد آخر mv old_name.py new_name.py # إعادة تسمية ملف mv *.py src/ # نقل كل ملفات Python إلى src
🔍 شرح الكود:

mv (Move) يقوم بوظيفتين: نقل ملف من مكان لآخر، وإعادة تسميته. نفس الأمر يُستخدم للحالتين — إذا كان الوجهة (destination) مجلداً موجوداً، يُنقل الملف إليه. إذا كان اسم ملف جديداً، يُعاد تسميته. mv أسرع من cp لأنه لا ينسخ البيانات بل يحرّر المؤشرات.

rm - حذف

rm file.txt # حذف ملف rm -r my_project/ # حذف مجلد ومحتوياته rm -rf my_project/ # حذف قسري دون تأكيد (خطير!) rm -i *.txt # حذف تفاعلي — يؤكد على كل ملف
🔍 شرح الكود:

rm (Remove) يحذف الملفات. الخيار -r (recursive) يحذف مجلداً وكل ما فيه. -f (force) يمنع أي تأكيدات — خطير جداً! -i (interactive) يسألك قبل حذف كل ملف. تحذير: rm -rf لا يسأل ولا يبقي شيئاً — استخدمه بحذر شديد. Linux لا يضع الملفات في سلة المهملات.

عرض الملفات

cat main.py # عرض محتوى الملف كاملاً less main.py # عرض تفاعلي — تصفّح بالأسهم، q للخروج head -n 10 main.py # أول 10 أسطر tail -n 20 main.py # آخر 20 سطر tail -f app.log # متابعة حية للملف (مثالي للـ logs)
🔍 شرح الكود:

cat يطبع الملف كاملاً — مفيد للملفات الصغيرة. less يفتح الملف في وضع تفاعلي للتصفح (أسهم، PageUp/Down، / للبحث، q للخروج). head يعرض أول N سطر. tail يعرض آخر N سطر. الخيار -f (follow) مع tail يتابع الملف حيّاً — ضروري لمراقبة سجلات التطبيقات.

4. Permissions - الصلاحيات

كل ملف ومجلد في Linux له صلاحيات تحدد من يمكنه القراءة والكتابة والتنفيذ. فهم الصلاحيات أساسي لأمان النظام ولتشغيل السكريبتات.

فهم rwx

ls -l script.py # -rwxr-xr-- 1 ahmed dev 1024 Jul 8 10:00 script.py # ^^^ ^^^ ^^^ # ||| ||| |||__ others (read only) # ||| |||___ group (read + execute) # |||________ user (read + write + execute) # |___________ type (- = file, d = directory)
🔍 شرح الكود:

الصلاحيات مقسمة لثلاث مجموعات: user (المالك)، group (المجموعة)، others (الآخرون). الحرف r = قراءة (4)، w = كتابة (2)، x = تنفيذ (1). الشرطة - تعني عدم وجود صلاحية. أول حرف يحدد النوع: - ملف عادي، d مجلد. في المثال: المالك لديه كل الصلاحيات، المجموعة تقرأ وتنفذ، والآخرون يقرؤون فقط.

chmod - تغيير الصلاحيات

# Numeric mode — الأكثر شيوعاً chmod 755 script.py # rwxr-xr-x — مالك كامل، الباقي قراءة+تنفيذ chmod 644 index.html # rw-r--r-- — مالك يكتب، الباقي يقرؤون فقط chmod 600 id_rsa # rw------- — فقط المالك يقرأ (للمفاتيح الخاصة) chmod 777 file # rwxrwxrwx — كل الصلاحيات للجميع (غير آمن!) # Symbolic mode chmod u+x script.py # أضف تنفيذ للمالك (user + execute) chmod g-w file.txt # احذف كتابة من المجموعة (group - write) chmod a+x script.py # أضف تنفيذ للكل (all + execute)
🔍 شرح الكود:

الأمر chmod (Change Mode) يغير صلاحيات الملف. الوضع الرقمي (numeric) يستخدم ثلاثة أرقام (0-7): الأول للمالك، الثاني للمجموعة، الثالث للآخرين. كل رقم هو مجموع: 4 (قراءة) + 2 (كتابة) + 1 (تنفيذ). 755 يعني المالك: 4+2+1=7، المجموعة: 4+1=5، الآخرون: 4+1=5. الوضع الرمزي (symbolic) يستخدم u (user) و g (group) و o (others) و a (all) مع + أو -.

chown - تغيير المالك

sudo chown ahmed:dev file.txt # غيّر المالك والمجموعة sudo chown ahmed file.txt # غيّر المالك فقط sudo chown -R ahmed:dev folder/ # غيّر لكل المحتويات (recursive)
🔍 شرح الكود:

chown (Change Owner) يغير مالك الملف و/أو مجموعته. يتطلب صلاحيات الجذر (sudo). الصيغة user:group تغير كليهما. الخيار -R يطبق التغيير على كل المجلدات الفرعية والملفات داخلها.

5. Processes - العمليات

إدارة العمليات (processes) ضرورية لمراقبة أداء التطبيقات وإيقافها عند الحاجة.

ps - عرض العمليات

ps # عمليات المستخدم الحالي ps aux # كل العمليات لكل المستخدمين ps aux | grep python # ابحث عن عمليات بايثون ps -ef # تنسيق كامل مع PID و PPID
🔍 شرح الكود:

ps (Process Status) يعرض العمليات الجارية. ps aux يعرض كل العمليات (a=all, u=user format, x=include daemons). ps -ef يعرض تنسيقاً كاملاً مع PID (رقم العملية) و PPID (رقم العملية الأب). مدمج مع grep لفلترة نتائج البحث.

top/htop - مراقبة حية

top # مراقبة العمليات في الوقت الحقيقي htop # نسخة مطورة وأجمل من top (ثبّتها أولاً) # داخل top: q = خروج, k = إنهاء عملية (أدخل PID), u = فلترة بمستخدم
🔍 شرح الكود:

top يعرض لوحة حية للعمليات مرتبة حسب استخدام CPU. اضغط q للخروج، k لإنهاء عملية (يطلب PID)، u لفلترة حسب مستخدم. htop بديل أفضل بألوان وواجهة تفاعلية وأزرار تحكم — ثبّته مع sudo apt install htop.

kill - إنهاء العمليات

kill 1234 # إنهاء عملية برقم PID 1234 (SIGTERM = طلب إنهاء) kill -9 1234 # SIGKILL — إنهاء قسري (لا يمكن للعملية تجاهله) kill -15 1234 # SIGTERM — إنهاء أنيق (يسمح بتنظيف الموارد) pkill -f python # إنهاء كل العمليات التي اسمها يحتوي python killall python # إنهاء كل عمليات python
🔍 شرح الكود:

kill يرسل إشارة (signal) لعملية ما. SIGTERM (15) هو الافتراضي — يطلب من العملية إنهاء نفسها بأناقة. SIGKILL (9) هو إنهاء قسري — نظام التشغيل يوقف العملية فوراً. pkill ينهي عمليات حسب الاسم. killall ينهي كل العمليات بهذا الاسم. استخدم SIGTERM أولاً، و SIGKILL كحل أخير.

التحكم بالعمليات في Terminal

python long_task.py # اضغط Ctrl+C — SIGINT, يوقف العملية فوراً # اضغط Ctrl+Z — SIGTSTP, يعلق العملية (مؤقتاً) bg # استئناف العملية المعلقة في الخلفية (background) fg # إعادة العملية للواجهة (foreground) jobs # عرض العمليات المعلقة والخلفية
🔍 شرح الكود:

Ctrl+C يرسل SIGINT — يوقف العملية. Ctrl+Z يرسل SIGTSTP — يعلق العملية (لا يوقفها). بعد تعليقها، bg يستأنفها في الخلفية (تستمر بالعمل دون أن تتحكم بالطرفية). fg يعيدها للواجهة. jobs يعرض قائمة بالعمليات المعلقة والخلفية مع أرقامها.

تشغيل عمليات في الخلفية

python app.py & # تشغيل في الخلفية مباشرة nohup python app.py & # تشغيل يقاوم إغلاق الطرفية nohup python app.py > app.log 2&1 & # مع تسجيل المخرجات disown %1 # افصل العملية عن الطرفية الحالية
🔍 شرح الكود:

علامة & في نهاية الأمر تشغّله في الخلفية — يظهر PID ثم يعود لك التحكم بالطرفية. nohup (No Hang Up) يجعل العملية تتجاهل إشارة SIGHUP التي ترسل عند إغلاق الطرفية. 2>&1 يدمج الخطأ المعياري (stderr) مع المخرجات المعيارية (stdout). disown يفصل العملية عن الطرفية الحالية.

6. Pipes and Redirection - الأنابيب وإعادة التوجيه

الأنابيب (pipes) وإعادة التوجيه (redirection) هي أقوى أدوات Linux — تسمح بدمج الأوامر بطرق إبداعية لمعالجة البيانات.

إعادة التوجيه (> و >> و 2>)

echo "print('hello')" > hello.py # اكتب النص في ملف (يكتب فوقه) echo "print('world')" >> hello.py # أضف النص لنهاية الملف python script.py 2> errors.log # أعد توجيه stderr إلى ملف python script.py > output.log 2&1 # ادمج stdout و stderr بنفس الملف python script.py > /dev/null 2&1 # تجاهل كل المخرجات (الثقب الأسود)
🔍 شرح الكود:

> يكتب المخرجات إلى ملف (يكتب فوق الملف). >> يضيف إلى نهاية الملف. الأرقام: 1 = stdout (المخرجات العادية)، 2 = stderr (الأخطاء). 2>&1 يوجه stderr إلى نفس وجهة stdout. /dev/null هو "الثقب الأسود" — أي شيء يُرسل إليه يُتجاهل.

الأنابيب (|)

ls -la | grep ".py" # ابحث عن ملفات Python ps aux | grep python | grep -v grep # عمليات بايثون بدون grep نفسه cat file.txt | sort | uniq # رتّب وأزل التكرارات history | awk '{print $2}' | sort | uniq -c | sort -rn | head # أكثر أوامرك استخداماً
🔍 شرح الكود:

علامة | (pipe) تأخذ مخرجات الأمر الأيسر وتغذيها كمدخل للأمر الأيمن. مثلاً ls -la | grep ".py": ls يسرد الملفات، ثم grep يفلتر فقط التي تحتوي ".py". يمكن ربط عدة أوامر — history | awk '{print $2}' | sort | uniq -c | sort -rn | head يعرض أكثر 10 أوامر استخداماً.

الإدخال من ملف (<)

python script.py < input.txt # استخدم input.txt كمدخل للسكريبت sort < unsorted.txt > sorted.txt # اقرأ من ملف واكتب النتيجة لآخر wc -l < data.csv # عد أسطر ملف CSV
🔍 شرح الكود:

< يوجه محتوى ملف كمدخل للأمر. مثلاً python script.py < input.txt يجعل محتوى input.txt متاحاً عبر sys.stdin. sort < unsorted.txt > sorted.txt يقرأ من ملف، يرتبه، ويكتب النتيجة لملف آخر — كل ذلك دون لمس الملف الأصلي.

7. Text Processing - معالجة النصوص

كمبرمج بايثون، غالباً ما تحتاج لتحليل ومعالجة النصوص مباشرة من CLI. هذه الأدوات تغني عن كتابة سكريبت بايثون للحالات البسيطة.

grep - البحث

grep "def" main.py # ابحث عن "def" في ملف grep -r "TODO" src/ # ابحث recursively في مجلد grep -i "error" app.log # بحث غير حساس لحالة الأحرف grep -n "import" *.py # أظهر رقم السطر مع النتيجة grep -c "def" *.py # عد مرات التكرار فقط grep -v "#" config.py # اعكس — أظهر الأسطر التي لا تحتوي # grep -l "class" *.py # أظهر أسماء الملفات فقط
🔍 شرح الكود:

grep (Global Regular Expression Print) يبحث عن نمط داخل ملف أو مخرجات أمر. الخيارات المهمة: -r بحث متكرر في مجلدات، -i تجاهل حالة الأحرف، -n إظهار رقم السطر، -c عد التكرارات، -v عكس النتيجة (عدم احتواء)، -l إظهار أسماء الملفات فقط. grep يقبل الـ regex — استخدمه للبحث عن أنماط معقدة.

sed - تحرير النص

sed 's/old/new/g' file.txt # استبدل old بـ new في كل الملف sed -i 's/foo/bar/g' file.txt # استبدل في نفس الملف (تعديل مباشر) sed '/^$/d' file.txt # احذف كل الأسطر الفارغة sed '3,10d' file.txt # احذف الأسطر من 3 إلى 10 sed 's/print(/logger.info(/g' app.py # استبدل print بـ logger.info
🔍 شرح الكود:

sed (Stream Editor) هو محرر نصوص غير تفاعلي. الصيغة الأساسية: s/pattern/replacement/flags. g (global) يعني استبدال كل التكرارات في السطر، ليس فقط الأولى. -i (in-place) يعدل الملف مباشرة (بدون نسخة احتياطية). /^$/d هو pattern: ^$ يعني سطر فارغ (بداية ثم نهاية)، d يعني delete.

awk - معالجة متقدمة

awk '{print $1, $3}' data.txt # اطبع العمود الأول والثالث awk '{print NR, $0}' file.txt # أضف رقم السطر قبل كل سطر awk '{sum += $1} END {print sum}' nums.txt # اجمع قيم العمود الأول awk '$3 > 50 {print $1, $3}' scores.txt # فلترة: اطبع إذا كان العمود 3 > 50 awk '{print $NF}' file.txt # اطبع آخر عمود awk -F ',' '{print $1, $2}' data.csv # حدد فاصل الفواصل (مثلاً CSV)
🔍 شرح الكود:

awk هي لغة معالجة نصوص كاملة. تقسم كل سطر إلى أعمدة ($1, $2, ... $NF = آخر عمود). NR = رقم السطر. $0 = السطر كاملاً. END تُنفذ بعد معالجة كل الأسطر. -F يحدد فاصل الأعمدة (افتراضياً: space/tab). مفيد جداً لتحليل ملفات CSV والسجلات.

wc, sort, uniq, cut

wc -l file.txt # عدد الأسطر wc -w file.txt # عدد الكلمات sort names.txt # ترتيب أبجدي sort -n numbers.txt # ترتيب عددي sort -rn numbers.txt # ترتيب عددي تنازلي uniq sorted.txt # إزالة الأسطر المتكررة (يحتاج sort أولاً) uniq -c sorted.txt # عد التكرارات cut -d',' -f1,3 data.csv # استخرج العمودين 1 و 3 من CSV cut -c1-10 file.txt # استخرج أول 10 أحرف من كل سطر
🔍 شرح الكود:

wc (Word Count) يحسب: -l أسطر، -w كلمات، -c بايتات. sort يرتب الأسطر: -n ترتيب عددي، -r عكس الترتيب. uniq يزيل التكرارات — لكنه يعمل فقط على الأسطر المتجاورة، لذا استخدم sort قبله. uniq -c يعد التكرارات. cut يستخرج أعمدة: -d فاصل، -f رقم العمود، -c أحرف.

8. SSH - الاتصال عن بعد

SSH (Secure Shell) يتيح لك الاتصال بأجهزة أخرى بشكل آمن — خوادم الإنتاج، GitHub، Raspberry Pi، وغيرها.

الاتصال الأساسي

ssh ahmed@192.168.1.100 # اتصل بخادم باسم مستخدم وعنوان IP ssh -p 2222 ahmed@server.com # استخدم منفذ مخصص (ليس 22 الافتراضي) ssh ahmed@server.com "ls -la" # نفذ أمراً عن بعد ثم اخرج ssh ahmed@server.com "python app.py" # شغّل سكريبت على الخادم
🔍 شرح الكود:

ssh user@host يفتح جلسة آمنة على الجهاز البعيد. يمكنك تحديد منفذ مختلف -p إذا كان الخادم يستخدم منفذاً غير 22. إذا أضفت أمراً بين علامتي اقتباس في النهاية، ينفذه ssh على الجهاز البعيد ثم يخرج — مثالي لأتمتة المهام عبر السكريبتات.

المفاتيح العمومية (SSH Keys)

ssh-keygen -t ed25519 # أنشئ زوج مفاتيح (الأفضل حالياً) ssh-keygen -t rsa -b 4096 # أو RSA 4096-bit cat ~/.ssh/id_ed25519.pub # اعرض المفتاح العمومي ssh-copy-id ahmed@server.com # انسخ المفتاح للخادم (للدخول بدون كلمة سر) ssh -i ~/.ssh/my_key ahmed@server.com # استخدم مفتاحاً محدداً
🔍 شرح الكود:

ssh-keygen ينشئ زوج مفاتيح — مفتاح خاص (id_ed25519) لا تشاركه أبداً، ومفتاح عمومي (id_ed25519.pub) تنسخه للخوادم. ssh-copy-id يضيف مفتاحك العمومي تلقائياً لملف ~/.ssh/authorized_keys على الخادم — بعدها تدخل بدون كلمة سر. ed25519 أكثر أماناً وأسرع من RSA.

scp - نقل الملفات

scp app.py ahmed@server.com:/home/ahmed/ # انسخ ملفاً للخادم scp -r project/ ahmed@server.com:/home/ahmed/ # انسخ مجلد كامل scp ahmed@server.com:/home/ahmed/data.csv . # اسحب ملفاً من الخادم
🔍 شرح الكود:

scp (Secure Copy) ينقل الملفات بين جهازك والخادم عبر SSH. الصيغة: scp [options] source destination. للملفات البعيدة استخدم user@host:path. -r للمجلدات. يمكنك النسخ من والى الخادم — ترتيب المصدر والوجهة يحدد الاتجاه.

9. Networking - الشبكات

أدوات الشبكات تسمح لك بتنزيل الملفات، اختبار الاتصالات، وفحص إعدادات الشبكة.

curl و wget - تنزيل البيانات

curl https://api.github.com/users/ahmed # GET request, يطبع النتيجة curl -o output.json https://api.example.com/data # احفظ الناتج بملف curl -s https://api.example.com # silent mode — بدون تقدم curl -I https://example.com # اعرض الهيدرز فقط (HEAD request) curl -X POST -d '{"name":"ahmed"}' -H "Content-Type: application/json" https://api.example.com/users wget https://example.com/file.zip # تنزيل ملف wget -c https://example.com/large.zip # استئناف تنزيل متقطع wget -r -l2 https://example.com/docs/ # تنزيل موقع كامل (عمق 2)
🔍 شرح الكود:

curl أداة قوية لنقل البيانات تدعم البروتوكولات المختلفة — ممتاز لاختبار APIs. -o يحفظ الناتج، -s silent، -I هيدرز فقط، -X يحدد نوع الطلب، -d يرسل بيانات. wget أبسط — ينزل الملفات بسرعة. -c يستأنف التنزيل، -r تنزيل متكرر للموقع.

ping - اختبار الاتصال

ping google.com # اختبر الاتصال بالإنترنت ping -c 4 8.8.8.8 # أرسل 4 حزم فقط ثم توقف ping -i 2 server.local # أرسل حزمة كل ثانيتين
🔍 شرح الكود:

ping يرسل حزم ICMP للجهاز المستهدف ويقيس زمن الاستجابة (latency). -c يحدد عدد الحزم التي سترسل. استخدمه لاختبار: هل الجهاز متصل؟ ما سرعة الاستجابة؟ هل هناك فقدان في الحزم؟ Ctrl+C يوقف ping.

netstat/ss و ip a

ss -tlnp # اعرض المنافذ المفتوحة (TCP, Listening, Numeric, PID) ss -tun # اتصالات TCP و UDP النشطة ss -tlnp | grep :8000 # هل المنفذ 8000 مشغول؟ ip a # اعرض جميع الواجهات وعناوين IP ip r # اعرض جدول التوجيه (routing) ip link # اعرض حالة الواجهات الشبكية
🔍 شرح الكود:

ss (Socket Statistics) هو البديل الحديث لـ netstat. -t TCP، -l المنافذ المنتظرة (listening)، -n إظهار الأرقام فقط (بدون أسماء)، -p إظهار الـ PID. ip a يعرض كل واجهات الشبكة مع عناوين IPv4 و IPv6 — استخدمه لمعرفة IP جهازك.

10. Bash Scripting Basics

كتابة سكريبتات Bash تسمح بأتمتة المهام المتكررة. كمبرمج بايثون، ستجد Bash مفيداً لتشغيل بايبلاينز سريعة دون كتابة سكريبت بايثون كامل.

الهيكل الأساسي (Shebang)

#!/bin/bash # هذا السكريبت يطبع "Hello World" echo "Hello World"
🔍 شرح الكود:

السطر الأول #!/bin/bash يُسمى shebang — يخبر النظام أي مفسر (interpreter) يستخدم لتنفيذ السكريبت. #! يتبعه مسار المفسر. /bin/bash هو Bash shell. بعد الحفظ، اجعل السكريبت executable: chmod +x script.sh، ثم شغّله: ./script.sh.

المتغيرات

#!/bin/bash # المتغيرات في Bash — بدون مسافات حول = name="Ahmed" echo "Hello, $name" # المتغيرات العددية والحساب count=10 count=$((count + 1)) echo "Count: $count" # وسائط السكريبت echo "Script: $0" # اسم السكريبت echo "First arg: $1" # أول وسيط ($2 = ثاني, $3 = ثالث) echo "All args: $@" # كل الوسائط echo "Number of args: $#" # عدد الوسائط
🔍 شرح الكود:

المتغيرات في Bash: اسم المتغير بدون مسافات حول =. استخدم $variable أو ${variable} للوصول للقيمة. العمليات الحسابية: $((expression)). $0 اسم السكريبت، $1 إلى $9 الوسائط، $@ كل الوسائط، $# عددها. اقتبس المتغيرات دائماً ("$var") لتجنب مشاكل المسافات.

حلقات for

#!/bin/bash # التكرار على قائمة for file in *.py; do echo "Processing $file" wc -l "$file" done # تكرار عددي for i in {1..5}; do echo "Iteration $i" done # تكرار على ملفات for name in ahmed sara khaled; do mkdir -p "projects/$name" done
🔍 شرح الكود:

حلقة for في Bash: for var in list; do ... done. *.py يتوسع إلى قائمة ملفات Python. {1..5} يولد الأرقام من 1 إلى 5. يمكنك التكرار على ملفات، أرقام، كلمات، أو مخرجات أمر (مثلاً: for file in $(ls *.txt)).

جمل if الشرطية

#!/bin/bash if [ -f "$1" ]; then echo "الملف $1 موجود" else echo "الملف $1 غير موجود" fi # شروط شائعة: # [-f file] — هل الملف موجود؟ # [-d dir] — هل المجلد موجود؟ # [-z str] — هل النص فارغ؟ # [a = b] — هل النصان متساويان؟ # [a != b] — هل النصان مختلفان؟ # [num1 -eq num2] — المساواة العددية
🔍 شرح الكود:

جمل if في Bash تختلف عن بايثون — الشرط بين قوسين مربعين [ ] مع مسافات حولهما. -f يختبر وجود ملف، -d يختبر وجود مجلد. المقارنة النصية: = و !=. المقارنة العددية: -eq (equal), -ne, -lt, -gt. fi تغلق if (if معكوسة).

جعل السكريبت executable

chmod +x script.sh # أضف صلاحية التنفيذ ./script.sh # شغّل السكريبت ./script.sh arg1 arg2 # مع وسائط python script.py # تشغيل بايثون — نفس المبدأ
🔍 شرح الكود:

chmod +x يضيف صلاحية التنفيذ (execute) للملف. ./script.sh يشغّله من الدليل الحالي — النقطة . تعني الدليل الحالي. إذا لم يكن الملف executable، سترى Permission denied. script.sh بدون ./ لن يعمل لأن الدليل الحالي ليس في PATH عادةً.

11. Environment Variables

متغيرات البيئة (Environment Variables) تخزن إعدادات النظام والتطبيقات — مثل مسارات Python، مفاتيح API، وإعدادات قاعدة البيانات.

export - تعريف متغيرات البيئة

export DB_HOST=localhost # عرّف متغير بيئة مؤقت export DB_USER=ahmed export DB_PASS=secret123 env | grep DB_ # تأكد من تعريف المتغيرات
🔍 شرح الكود:

export VAR=value يعرّف متغير بيئة متاحاً للعمليات الابنة (child processes) — أي السكريبتات والبرامج التي تشغلها من هذه الطرفية. بدون export، المتغير يكون محلياً فقط. env يعرض كل متغيرات البيئة الحالية.

استخدام متغيرات البيئة في بايثون

#!/usr/bin/env python import os db_host = os.environ.get('DB_HOST', 'localhost') db_user = os.environ.get('DB_USER') db_pass = os.environ.get('DB_PASS') print(f"Connecting to {db_host} as {db_user}")
🔍 شرح الكود:

في بايثون، os.environ هو قاموس يحتوي كل متغيرات البيئة. استخدم .get('KEY', default) لقراءة متغير — الثاني هو القيمة الافتراضية إذا لم يكن المتغير موجوداً. هذا النمط (قراءة الإعدادات من البيئة) هو الـ 12-Factor App methodology لتطبيقات قابلة للنقل.

PATH - أين يبحث النظام عن الأوامر؟

echo $PATH # اعرض مسارات البحث عن الأوامر # /usr/local/bin:/usr/bin:/bin:/usr/games # أضف مجلداً لـ PATH (مثلاً لمشاريعك) export PATH=$PATH:/home/ahmed/my_scripts # أي أمر executable في هذا المجلد سيكون متاحاً من أي مكان which python3 # أين يوجد python3؟ which pip # أين يوجد pip؟
🔍 شرح الكود:

PATH هو متغير بيئة خاص — قائمة من المسارات يفصلها :. عندما تكتب أمراً مثل python3، يبحث النظام في كل مسار في PATH من اليسار لليمين حتى يجد الملف. which يخبرك أي مسار سيستخدم. أضف مجلد سكريبتاتك لـ PATH عشان تشغّلهم من أي مكان.

~/.bashrc و ~/.bash_profile

# افتح ملف bashrc للتحرير nano ~/.bashrc # أضف هذه الأسطر في النهاية ثم احفظ واخرج export EDITOR=nano export PYTHONSTARTUP=~/.pythonrc alias ll='ls -la' alias gs='git status' alias gp='git pull' alias venv='python -m venv venv && source venv/bin/activate' # طبّق التغييرات دون إعادة تشغيل الطرفية source ~/.bashrc
🔍 شرح الكود:

ملف ~/.bashrc يُنفذ في كل مرة تفتح فيها terminal جديد. أضف فيه: export لمتغيرات البيئة الدائمة، alias لاختصارات الأوامر. alias ll='ls -la' يعني أن كتابة ll تنفذ ls -la. source ~/.bashrc يعيد تحميل الملف في الجلسة الحالية. ~/.bash_profile يُنفذ مرة واحدة عند الدخول (login shell).

أمثلة على Aliases مفيدة لمطور بايثون

# أضفها في ~/.bashrc alias py=python3 alias pir='pip install -r requirements.txt' alias pyt='pytest -v' alias act='source venv/bin/activate' alias djs='python manage.py runserver' alias his='history | grep'
🔍 شرح الكود:

الـ aliases توفر وقتاً كبيراً. py=python3 — اختصار لتشغيل بايثون. pir ينصب المتطلبات. pyt يشغل الاختبارات. act ينشط البيئة الافتراضية. djs يشغل خادم Django. his يبحث في التاريخ — مثلاً his docker يعرض كل أوامر docker التي كتبتها.

💡 نصيحة: استخدم export SECRET_KEY=$(cat /path/to/secret) لقراءة المفاتيح من ملفات بدلاً من كتابتها مباشرة في bashrc. ولا تشارك أبداً ملف bashrc الذي يحتوي مفاتيح حقيقية على GitHub — استخدم ملف .env مع python-dotenv بدلاً من ذلك.

📝 ملخص أوامر Linux

في هذا الدليل تعلمت:

  • التنقل: pwd, ls, cd
  • الملفات: touch, mkdir, cp, mv, rm
  • الصلاحيات: chmod 755, chmod u+x, chown
  • العمليات: ps, top, kill, &, nohup
  • الأنابيب: |, >, >>, 2>&1, /dev/null
  • النصوص: grep, sed, awk, sort, uniq
  • الاتصال: ssh, ssh-keygen, scp
  • الشبكات: curl, wget, ping, ss, ip a
  • السكريبتات: #!/bin/bash, المتغيرات، for, if
  • متغيرات البيئة: export, PATH, alias, ~/.bashrc
⚠️ تحذيرات مهمة: rm -rf / بدون sudo لن تعمل، لكن rm -rf ~ أو rm -rf . في المكان الخطأ كارثية. دائماً تحقق من الدليل الحالي (pwd) قبل حذف مجلد. استخدم -i (interactive) مع rm و cp و mv إذا كنت غير متأكد. لا تشغل sudo مع أوامر لا تفهمها تماماً.
